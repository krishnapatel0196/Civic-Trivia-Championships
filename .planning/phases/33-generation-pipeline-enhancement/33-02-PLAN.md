    **CLI Arguments Parser (function parseArgs):**
    - `--collection` (required): Collection slug matching JSON filename (e.g., "fremont-ca" -> "fremont-ca-questions.json"). Special case: "federal" -> "questions.json".
    - `--target` (required): Target total question count for the collection (e.g., 90).
    - `--dry-run` (optional): Show gap analysis only, don't generate.
    - `--limit` (optional): Maximum number of questions to generate in this run (useful for testing or incremental generation).

    **Collection Data Loader (function loadCollectionData):**
    1. Map collection slug to JSON file path:
       - "federal" -> `src/data/questions.json`
       - Others -> `src/data/{slug}-questions.json` (e.g., "fremont-ca" -> "fremont-ca-questions.json")
       - Map slug to display name using a const lookup:
         ```
         const COLLECTION_NAMES: Record<string, string> = {
           'federal': 'Federal Civics',
           'bloomington-in': 'Bloomington, IN',
           'los-angeles-ca': 'Los Angeles, CA',
           'indiana': 'Indiana',
           'california': 'California',
           'fremont-ca': 'Fremont, CA',
         };
         ```
    2. Read JSON file, parse topics array and questions array
    3. Return `{ collectionName, topics: TopicConfig[], existingQuestions: GeneratedQuestion[] }`

    **Existing Questions Loader (function loadExistingQuestionsForDedup):**
    1. Query the database for ALL active questions across ALL collections (same query as scan-duplicates.ts: join questions with collections via question_collections)
    2. Return as `QuestionForDedup[]` (externalId, text, options, correctAnswer, collections array, qualityScore)
    3. This provides the full corpus for cross-collection dedup checks

    **Prompt Builder (function buildGenerationPrompt):**
    Build a Claude prompt for generating ONE question at a time.

    System prompt (include locale context):
    ```
    You are generating civic trivia questions for the "{collectionName}" collection.

    Tone and style:
    - Questions should feel fun and useful to know, not like a test or quiz
    - Think "dinner party test": would knowing this answer be worth sharing?
    - Easy questions are welcome -- "Who is your mayor?" feels fair and memorable
    - Be strictly nonpartisan: present facts only, no political labels
    - For time-sensitive facts, note "as of {date}" in the explanation

    Quality requirements:
    - 4 answer options, exactly one correct
    - Options must be clearly distinguishable (no "all of the above", no overlapping ranges)
    - Explanation should be 1-2 sentences explaining WHY the answer matters, not just restating it
    - Source must be an authoritative URL (.gov, .edu, official city/county/state websites, Wikipedia for general facts)
    - No phone numbers, street addresses, or pure lookup trivia with no civic value

    {config.localContext}
    ```

    User prompt (parameterized by topic and difficulty):
    ```
    Generate a {difficulty} civic trivia question about "{topicName}" for {collectionName}.

    Topic description: {topicDescription}

    Difficulty guidelines:
    - easy: Common knowledge most residents would know or could reasonably guess
    - medium: Requires some civic awareness but not specialized knowledge
    - hard: Requires deeper knowledge of civic systems, history, or processes

    {retryFeedback -- empty on first attempt, includes violation details on retry}

    Return ONLY valid JSON (no markdown code blocks, no extra text):
    {
      "text": "The question text?",
      "options": ["Option A", "Option B", "Option C", "Option D"],
      "correctAnswer": 0,
      "explanation": "Brief explanation of why this answer matters.",
      "difficulty": "{difficulty}",
      "topicCategory": "{topicSlug}",
      "source": {
        "name": "Source Name",
        "url": "https://authoritative-source.gov/..."
      },
      "expiresAt": null
    }
    ```

    For retry prompts when quality violations occur, APPEND a focused feedback section:
    ```
    IMPORTANT: Your previous attempt had these issues:
    {violations.map(v => `- ${v.rule}: ${v.message}`).join('\n')}
    Please generate a NEW question that avoids these issues. Do not revise the previous question.
    ```

    For retry prompts when duplicate detected, APPEND:
    ```
    IMPORTANT: Your previous attempt was too similar to an existing question. Please generate a completely different question about this topic. Choose a different angle or subtopic.
    ```

    **Single Question Generator (async function generateOneQuestion):**
    Parameters: `(client: Anthropic, prompt: string, config: GenerationConfig, slot: GenerationSlot, existingForDedup: QuestionForDedup[], batchQuestions: GeneratedQuestion[], batchEmbeddings: Map<string, number[]>, sourceTracker: SourceTracker, hierarchy: CollectionHierarchy, attempt: number = 1)`

    Logic (the core generation loop):
    1. Call Claude API:
       ```
       const response = await client.messages.create({
         model: 'claude-sonnet-4-5-20250929',
         max_tokens: 1024,
         system: systemPrompt,
         messages: [{ role: 'user', content: userPrompt }],
       });
       ```
    2. Parse JSON from response (use regex pattern from generateLearningContent.ts: `contentText.match(/\{[\s\S]*\}/)`)
    3. Validate structure: must have text, options (array of 4), correctAnswer (0-3), explanation, difficulty, topicCategory, source with name and url
    4. Run quality audit: `const audit = await auditQuestion(parsed, { skipUrlCheck: true })` -- skip URL check during generation (sources are AI-generated, URL validity is advisory)

    5. **Handle blocking violations:**
       - If `audit.hasBlockingViolations` AND attempt < 3:
         Build retry prompt with violation feedback, recurse with attempt + 1
       - If `audit.hasBlockingViolations` AND attempt >= 3:
         Return `{ question: parsed, status: 'skipped', attempts: attempt, violations: audit.violations }`

    6. **Handle advisory violations:**
       - If `audit.hasAdvisoryOnly` AND attempt === 1:
         Build retry prompt with advisory feedback, recurse with attempt + 1 (one retry for advisory)
       - If `audit.hasAdvisoryOnly` AND attempt > 1:
         Accept the question (advisory violations are non-blocking after one retry)

    7. **Semantic dedup check (only if quality passed):**
       Call `hierarchy.checkDuplicate(parsed.text, parsed.options, config.collectionName, existingForDedup, batchEmbeddings)`
       - If duplicate AND attempt < 3: retry with duplicate feedback
       - If duplicate AND attempt >= 3: return `{ status: 'duplicate', ... }`

    8. **Source diversity check:**
       Call `sourceTracker.checkSource(parsed.source)`
       - If warning: log it, attach to result. Still accept (soft cap).
       - Call `sourceTracker.recordSource(parsed.source)`

    9. **If accepted:**
       - Generate an externalId: use pattern `{collectionPrefix}-{nextNumber}` where collectionPrefix is derived from slug (e.g., "fre" for fremont-ca, "bloom" for bloomington-in, "la" for los-angeles-ca, "ind" for indiana, "cal" for california, "fed" for federal) and nextNumber is max existing + batch count + 1
       - Add embedding to batchEmbeddings for intra-batch dedup
       - Return `{ question: parsed, status: 'accepted', attempts: attempt, violations: audit.violations, sourceWarning }`

    **Main Orchestrator (async function main):**
    1. Parse args, validate required flags
    2. Check env vars: `ANTHROPIC_API_KEY`, `OPENAI_API_KEY`
    3. Load collection data from JSON file
    4. Load all existing questions from database for dedup
    5. Initialize services:
       - `const client = new Anthropic()`
       - `const embeddingService = new OpenAIEmbeddingService({ apiKey: process.env.OPENAI_API_KEY! })`
       - `const gapAnalyzer = new GapAnalyzer()`
       - `const sourceTracker = new SourceTracker(existingQuestions)`
       - `const hierarchy = new CollectionHierarchy(embeddingService)`
    6. Run gap analysis: `const gaps = gapAnalyzer.analyzeGaps(existingQuestions, targetCount, topics)`
    7. Print gap analysis summary via `gapAnalyzer.summarize(gaps)`
    8. If `--dry-run`: print summary and exit
    9. Apply `--limit` to generationPlan if provided
    10. Generate questions one at a time:
        ```
        const batchQuestions: GeneratedQuestion[] = [];
        const batchEmbeddings = new Map<string, number[]>();
        const results: GenerationResult[] = [];

        for (let i = 0; i < generationPlan.length; i++) {
          const slot = generationPlan[i];
          console.log(`\n[${i + 1}/${generationPlan.length}] Generating ${slot.difficulty} question for "${slot.topic}"...`);

          const result = await generateOneQuestion(
            client, prompt, config, slot,
            existingForDedup, batchQuestions, batchEmbeddings,
            sourceTracker, hierarchy
          );

          results.push(result);

          if (result.status === 'accepted') {
            batchQuestions.push(result.question);
            console.log(`  Accepted: ${result.question.text.substring(0, 60)}... (${result.attempts} attempt(s))`);
          } else {
            console.log(`  ${result.status}: ${result.violations.map(v => v.rule).join(', ')} (${result.attempts} attempt(s))`);
          }

          // Rate limiting: 1 second between generation requests
          await new Promise(resolve => setTimeout(resolve, 1000));
        }
        ```
    11. Save embedding cache: `embeddingService.saveCache()`
    12. Build GenerationReport from results
    13. Write accepted questions to JSON output file:
        ```
        const timestamp = new Date().toISOString().replace(/[:.]/g, '-').slice(0, -5);
        const outputPath = join(process.cwd(), `generated-questions-${config.collectionSlug}-${timestamp}.json`);
        ```
        Write the output in the SAME format as the collection JSON files (with topics array and questions array) so it can be reviewed and merged.
    14. Write report to a separate file:
        ```
        const reportPath = join(process.cwd(), `generation-report-${config.collectionSlug}-${timestamp}.json`);
        ```
    15. Print final summary to console:
        ```
        console.log('\n' + '='.repeat(60));
        console.log('GENERATION COMPLETE');
        console.log('='.repeat(60));
        console.log(`Collection: ${config.collectionName}`);
        console.log(`Accepted: ${accepted} / ${totalAttempted}`);
        console.log(`Skipped (quality): ${skipped}`);
        console.log(`Skipped (duplicate): ${duplicates}`);
        console.log(`\nOutput: ${outputPath}`);
        console.log(`Report: ${reportPath}`);
        console.log('='.repeat(60));
        ```

    **Important implementation notes:**
    - Follow the established pattern from generateLearningContent.ts for Claude API calls and JSON parsing
    - Follow the established pattern from scan-duplicates.ts for database queries
    - Use serial generation (1 question at a time with 1-second delay) per RESEARCH.md recommendation
    - Output to JSON files, NOT directly to database (per RESEARCH.md pitfall 5 and CONTEXT.md "generate summary report")
    - The script should be resilient: if one question fails, log and continue to next
    - The db import may need `../env.js` imported first for dotenv (check scan-duplicates.ts pattern)
