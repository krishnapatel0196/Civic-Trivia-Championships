    2. Create `backend/src/services/generation/GapAnalyzer.ts`:

       Import `TopicConfig`, `CollectionGaps`, `GenerationSlot`, `DifficultyDistribution`, `DEFAULT_DIFFICULTY_DISTRIBUTION` from `./types.js`.

       Export class `GapAnalyzer` with these methods:

       - `analyzeGaps(existingQuestions: Array<{ topicCategory: string; difficulty: string }>, targetCount: number, topics: TopicConfig[], difficultyDistribution?: DifficultyDistribution): CollectionGaps`

         Logic:
         1. Count current topic distribution (Map of topic slug -> count)
         2. Count current difficulty distribution (Map of difficulty -> count)
         3. Calculate topic targets: for each topic, target is `max(minQuestionsPerTopic, floor(targetCount / topics.length))` where minQuestionsPerTopic defaults to 10
         4. Calculate difficulty targets: `easy = round(targetCount * distribution.easy)`, etc.
         5. Compute topic gaps: for each topic, `max(0, target - current)`
         6. Compute difficulty gaps: for each difficulty, `max(0, target - current)`
         7. Call `redistributeIfNeeded()` to handle unreachable topics
         8. Build generationPlan by interleaving topics and difficulties:
            - Create GenerationSlot entries that satisfy BOTH topic AND difficulty gaps
            - Prioritize the largest gaps first
            - Use a greedy approach: pick the topic with the biggest gap, assign the difficulty with the biggest gap, decrement both, repeat
         9. Return CollectionGaps with topicGaps, difficultyGaps, totalNeeded, and generationPlan

       - `private redistributeIfNeeded(topicGaps: Map<string, number>, topics: TopicConfig[]): void`

         If any topic needs more questions than half the total gap, it's likely unreachable. Redistribute its allocation proportionally among remaining topics. Log a warning when redistribution occurs. This handles the CONTEXT.md decision: "When a topic can't reach the minimum count, redistribute."

       - `summarize(gaps: CollectionGaps): string`

         Returns a human-readable summary of the gap analysis, formatted for console output. Show current vs target for each topic and difficulty level.

       Use `minQuestionsPerTopic = 10` as a private class constant.

    1. Create `backend/src/services/generation/SourceTracker.ts`:

       Export class `SourceTracker` that tracks source domain usage across a collection and enforces soft diversity caps.

       Constructor: `constructor(existingQuestions: Array<{ source: { name: string; url: string } }>)`
       - Count existing source usage by domain (extract hostname, strip `www.` prefix)
       - Store as `Map<string, number>` (domain -> count)
       - Track totalQuestions count

       Methods:

       - `private getSourceKey(source: { name: string; url: string }): string`
         Extract hostname from URL, remove `www.` prefix. Fall back to full URL string if URL parsing fails. This is Claude's discretion decision: track by domain for simplicity and true diversity.

       - `checkSource(source: { name: string; url: string }): { allowed: boolean; warning?: string }`
         Calculate current percentage for this source's domain.
         Calculate adaptive cap using `getAdaptiveCap()`.
         If percentage >= cap: return `{ allowed: true, warning: "Source X at Y%, exceeds Z% cap" }`.
         Per CONTEXT.md: 15% is a SOFT WARNING, not a hard block. Always return `allowed: true`.
         If under cap: return `{ allowed: true }` (no warning).

       - `private getAdaptiveCap(): number`
         Scale cap based on number of unique source domains:
         - 5+ unique sources: 0.15 (15% standard)
         - 4 unique sources: 0.20 (20%)
         - 3 or fewer: 0.25 (25%)
         Per CONTEXT.md: "For locales with limited online sources, scale the cap with availability."

       - `recordSource(source: { name: string; url: string }): void`
         Increment count for this source's domain. Increment totalQuestions.

       - `getSummary(): Array<{ sourceKey: string; count: number; percentage: number }>`
         Return array sorted by count descending. Each entry shows domain, count, and percentage of total.

    2. Create `backend/src/services/generation/CollectionHierarchy.ts`:

       Import `COLLECTION_HIERARCHY, TIER_RANK, CollectionTier, QuestionForDedup` from `../embeddings/types.js`.
       Import `SemanticDupDetector` from `../embeddings/SemanticDupDetector.js`.
       Import `OpenAIEmbeddingService` from `../embeddings/OpenAIEmbeddingService.js`.

       Export interface `DuplicateCheckResult`:
       - `isDuplicate: boolean`
       - `duplicateOf?: string` (externalId of matching question)
       - `matchScore?: number` (similarity score for audit trail)
       - `reason?: string` (human-readable explanation)

       Export class `CollectionHierarchy` with:

       Constructor: `constructor(private embeddingService: OpenAIEmbeddingService)`

       Methods:

       - `async checkDuplicate(questionText: string, questionOptions: string[], targetCollection: string, existingQuestions: Array<{ externalId: string; text: string; options: string[]; collections: string[]; embedding?: number[] }>, batchEmbeddings?: Map<string, number[]>): Promise<DuplicateCheckResult>`

         Logic:
         1. Prepare text using `SemanticDupDetector.prepareTextForEmbedding({ text: questionText, options: questionOptions, ... })`
         2. Generate embedding for new question via `this.embeddingService.embed(text)`
         3. Determine target collection tier from `COLLECTION_HIERARCHY`
         4. For each existing question:
            a. Get or compute its embedding (from provided embedding or via embeddingService)
            b. Compute cosine similarity via `SemanticDupDetector.cosineSimilarity()`
            c. If similarity > 0.85 (NEAR_DUPLICATE_THRESHOLD, same as Phase 31-32):
               - Check hierarchy: what tier are the existing question's collections?
               - REJECT if existing is in same collection or parent collection (higher tier)
               - ALLOW if existing is only in sibling or child collections
         5. Also check batchEmbeddings (Map of "batch-N" -> embedding) for intra-batch dedup. Intra-batch duplicates always rejected.
         6. Return `{ isDuplicate, duplicateOf, matchScore, reason }`

         Use 0.85 threshold constant (NEAR_DUPLICATE_THRESHOLD) at top of file. Same as Phase 31-32 scanner.

       - `static getCollectionTier(collectionName: string): CollectionTier | undefined`
         Wrapper around COLLECTION_HIERARCHY lookup. Returns undefined for unknown collections.

       - `static isParentOrSame(existingCollection: string, targetCollection: string): boolean`
         Returns true if existingCollection is the same as or a parent of targetCollection.
         Parent means higher TIER_RANK. Same means identical string.
