   import { createClient } from '@supabase/supabase-js';

<task type="auto">
  <name>Task 2: Replace auth.ts middleware entirely</name>
  <files>backend/src/middleware/auth.ts</files>
  <action>
Replace the entire contents of `backend/src/middleware/auth.ts`. The new file must:

1. **Remove all old imports** — no more `tokenUtils.ts`, `isTokenBlacklisted`, `verifyAccessToken`, `TokenPayload`. These are dead code paths after this change.

2. **Express Request type extension** — replace `req.user?: TokenPayload` with:
   ```typescript
   declare global {
     namespace Express {
       interface Request {
         userId?: string;       // UUID from Supabase JWT sub claim
         accessToken?: string;  // Raw JWT for downstream use
       }
     }
   }
   ```

3. **Module-level SECRET_KEY** — encode once, reuse on every request:
   ```typescript
   const SECRET_KEY = new TextEncoder().encode(process.env.SUPABASE_JWT_SECRET);
   ```

4. **requireAuth** — copy verbatim from integration guide section 1:
   - Extract token from `Authorization: Bearer <token>` header (use `.slice(7)`)
   - No token → 401 `{ error: 'Missing token' }`
   - `jwtVerify(token, SECRET_KEY, { issuer: \`${process.env.SUPABASE_URL}/auth/v1\`, audience: 'authenticated' })`
   - CRITICAL: issuer MUST be `${SUPABASE_URL}/auth/v1` — not just SUPABASE_URL
   - Set `req.userId = payload.sub as string` and `req.accessToken = token`
   - Catch → 401 `{ error: 'Invalid or expired token' }`

5. **Export alias** — `export { requireAuth as authenticateToken }` so existing callers that import `authenticateToken` continue to work without changes. This is a temporary bridge — callers will be updated in Plan 02.

6. **optionalAuth** — same jwtVerify logic but:
   - No token → set `req.userId = undefined`, call `next()` (no error)
   - Invalid token → set `req.userId = undefined`, call `next()` (no error)
   - Valid token → set `req.userId` and `req.accessToken`, call `next()`
   - NO blacklist check (Supabase JWTs are stateless)

7. **requireConnected** — per integration guide section 2:
   - Import `supabaseAdmin` from `../config/supabase.js`
   - Query: `(supabaseAdmin as any).schema('connect').from('connected_profiles').select('verification_status').eq('user_id', req.userId!).maybeSingle()`
   - CRITICAL: Cast to `any` before `.schema('connect')` — the `connect` schema is NOT in database.types.ts
   - No data or `verification_status !== 'verified'` → 403 `{ error: 'Connected account required' }`

8. **requireAdmin** — use `admin_users` table (simpler than user_roles join, correctly typed in database.types.ts):
   - Guard: if `!req.userId` → 401 `{ error: 'Authentication required' }`
   - Query: `supabaseAdmin.from('admin_users').select('user_id').eq('user_id', req.userId).maybeSingle()`
   - No data → 403 `{ error: 'Admin required' }`

All middleware functions must have explicit return type `Promise<void>` and use `res.status().json(); return;` pattern (no `return res.status()` — Express types don't like that).
