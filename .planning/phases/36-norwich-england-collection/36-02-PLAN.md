**Step 2: Fetch RAG sources**

Run from the `backend/` directory:
```bash
npx tsx src/scripts/content-generation/generate-locale-questions.ts --locale norwich-uk --fetch-sources --dry-run --batch 1
```

This fetches source documents and runs a test batch. Check that:
- Sources download successfully (some 404s are expected; the script continues)
- Batch 1 generates questions with UK terminology
- No US-specific terms in generated output

**Step 3: Run full generation**

Run the full generation pipeline:
```bash
npx tsx src/scripts/content-generation/generate-locale-questions.ts --locale norwich-uk --dry-run
```

Use `--dry-run` first to validate output quality before seeding.

Review the console output for:
- Total questions generated (target: 90+ with overshoot)
- Quality validation pass rate
- Topic distribution across all 8 categories
- No US-centric terms in generated questions

If dry-run looks good, run without `--dry-run` to seed directly:
```bash
npx tsx src/scripts/content-generation/generate-locale-questions.ts --locale norwich-uk
```

**IMPORTANT:** The generation script seeds questions with `status: 'draft'` when run without `--dry-run`. The questions still need to be inserted via seed-community.ts or activated through admin.

**Alternative approach if direct seeding is used:** The generation script (without --dry-run) seeds directly to DB and creates the questions. In that case, skip the seed-community step and just verify questions exist in DB.

**Step 4: Post-generation attribution check**

After generation, scan the output for two-tier attribution errors. Search the generated JSON for patterns:
- Questions mentioning "Norwich City Council" + county-tier keywords (roads, schools, education, social care, library, fire)
- Questions mentioning "Norfolk County Council" + city-tier keywords (housing, planning, leisure, bin collection, council tax)

Log any warnings. These are flagged for review, not auto-rejected.

If using --dry-run approach: The generated questions need to be saved to `backend/src/data/norwich-uk-questions.json`. The generation script in dry-run mode outputs to console/reports but does not write the data file. In that case, run without --dry-run which seeds directly to DB.

**Step 1: Add collection card image**

Download or create a Norwich Cathedral image for the collection card. The image should be:
- JPEG format, ~800x600px (matching other collection images)
- Norwich Cathedral prominently framed (the city's most iconic landmark, per CONTEXT.md)
- Saved to `frontend/public/images/collections/norwich-uk.jpg`

If a direct download is not possible via CLI, create a placeholder image or use an available free-license image. The user can replace it later.

Options for sourcing:
1. Use `curl` to download a Creative Commons Norwich Cathedral image from Wikimedia Commons
2. If no suitable image is available via CLI, create a solid-color placeholder (deep forest green #1B4332) with text "Norwich" so the collection card is not broken

**Step 2: Ensure collection row exists in database**

Run the base seed to create the Norwich collection row (if not already created):
```bash
cd backend && npx tsx src/db/seed/seed.ts
```

This is idempotent (ON CONFLICT DO NOTHING). It creates the norwich-uk collection row with `isActive: false`.

**Step 3: Seed questions (if using data file approach)**

If questions were generated via --dry-run and saved to the JSON data file:
```bash
cd backend && npx tsx src/db/seed/seed-community.ts
```

This reads norwich-uk-questions.json and inserts questions to DB.

If questions were already seeded directly by the generation script (non-dry-run), skip this step.

**Step 4: Activate the collection**

Activate the Norwich collection so it appears in the picker. Run a direct SQL update:
```bash
cd backend && npx tsx -e "
import 'dotenv/config';
import { db } from './src/db/index.js';
import { collections } from './src/db/schema.js';
import { eq } from 'drizzle-orm';

const result = await db.update(collections).set({ isActive: true }).where(eq(collections.slug, 'norwich-uk')).returning();
console.log('Activated:', result.length, 'row(s)');
console.log(result[0]);
process.exit(0);
"
```

**Step 5: Verify end-to-end**

Run verification queries:
```sql
-- Collection exists and is active
SELECT id, name, slug, is_active, theme_color FROM civic_trivia.collections WHERE slug = 'norwich-uk';

-- Question count
SELECT COUNT(*) as total FROM civic_trivia.questions WHERE external_id LIKE 'nor-%';

-- Topic distribution
SELECT subcategory, COUNT(*) as count FROM civic_trivia.questions WHERE external_id LIKE 'nor-%' GROUP BY subcategory ORDER BY count DESC;

-- Sample questions (check UK terminology)
SELECT external_id, text FROM civic_trivia.questions WHERE external_id LIKE 'nor-%' LIMIT 5;

-- Collection-question links
SELECT COUNT(*) FROM civic_trivia.collection_questions cq JOIN civic_trivia.collections c ON c.id = cq.collection_id WHERE c.slug = 'norwich-uk';
```
