**Capture priorLevel when XP data loads:** Add a useEffect that stores the level once:
```typescript
useEffect(() => {
  // Capture priorLevel when XP loads on idle screen (before game starts)
  // Reset when a new game starts so repeated plays detect the CURRENT pre-game level
  if (state.phase === 'idle' && xpData?.level !== undefined) {
    setPriorLevel(xpData.level);
  }
}, [state.phase, xpData?.level]);
```

This approach: when the player returns to idle after a game (Play Again), xpData refetches (userId doesn't change but phase re-entering idle triggers the effect). Actually xpData only refetches if userId changes. Since the user doesn't change between games, priorLevel from the PREVIOUS game's award would be stored — which is correct. The level after the last game becomes the priorLevel before the next game.

But there's a subtlety: after the first game, progression.xp.level may be higher than xpData (which was fetched at game start). So for subsequent games, we need to update priorLevel from xpData when idle returns AND xpData has reloaded. Since `usePlayerXp` doesn't auto-refetch on idle re-entry, we need a different approach.

Better approach: store priorLevel from xpData when it first loads, and reset it when `state.phase` transitions to `starting` (game just started). Then on the NEXT idle, the xpData will have been re-fetched (because after game complete, update priorLevel from progression result):

Actually simplest: capture priorLevel from `xpData.level` any time phase === 'idle' AND xpData exists. Since `usePlayerXp` only fetches once (on userId change), this will store the pre-game level correctly for the first game. For subsequent games (Play Again), the player stays on the same userId so xpData doesn't refetch — but that's OK because after game completion in Plan 05, we'll update priorLevel from `progression.xp.level` (the post-award level becomes the new priorLevel).

```typescript
// Capture priorLevel when idle + xpData available
useEffect(() => {
  if (state.phase === 'idle' && xpData !== null && priorLevel === null) {
    setPriorLevel(xpData.level);
  }
}, [state.phase, xpData, priorLevel]);

// Reset priorLevel when game completes so Plan 05 can set it from the award response
// (priorLevel will be set to progression.xp.level after results, ready for next game)
// This reset + the Plan 05 update is how priorLevel stays accurate across replays
```
