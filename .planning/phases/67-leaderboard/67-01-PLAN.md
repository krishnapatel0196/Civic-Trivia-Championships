<task type="auto">
  <name>Task 2: Create GET /api/leaderboard route with caching</name>
  <files>backend/src/routes/leaderboard.ts, backend/src/server.ts</files>
  <action>
Using the schema findings from Task 1, build the leaderboard backend route.

Create `backend/src/routes/leaderboard.ts`:

1. **Route:** `GET /` (mounted at `/api/leaderboard` in server.ts)
   - Query params: `tab` (string, default `all_time`, valid values: `all_time` | `this_week`), `userId` (optional UUID string)
   - No auth middleware (public endpoint)

2. **Response shape:**
```typescript
{
  tab: 'all_time' | 'this_week';
  entries: Array<{
    rank: number;
    user_id: string;
    username: string;
    tier: string;       // 'inform' | 'connected' | 'empowered'
    level: number;
    total_xp: number;
  }>;
  userRank: {
    rank: number;
    username: string;
    tier: string;
    level: number;
    total_xp: number;
    gap_to_next: number; // XP difference to the player ranked one above
  } | null;             // null if userId not provided or user has no XP
  generatedAt: string;  // ISO timestamp
}
```

3. **Caching:** Use `storageFactory.getStorage()` with 5-minute (300 second) TTL.
   - Cache key format: `leaderboard:${tab}` for the entries list (shared across all visitors)
   - For userRank: compute separately from the cached entries if possible, or use a separate cache key `leaderboard:rank:${tab}:${userId}`
   - Parse cached value with JSON.parse, store with JSON.stringify

4. **All Time query logic:**
   - Query the Supabase table/function identified in Task 1 for top 25 users ordered by total_xp DESC
   - Each entry needs: user_id, username, tier, level, total_xp
   - If using a SECURITY DEFINER function: call via `supabaseAdmin.rpc('get_leaderboard', { p_tab: 'all_time', p_limit: 25 })`
   - If using direct table access: `supabaseAdmin.from('table').select('...').order('total_xp', { ascending: false }).limit(25)`

5. **This Week query logic:**
   - SUM XP transactions from the last 7 days (rolling, NOT calendar week)
   - `WHERE created_at > NOW() - INTERVAL '7 days'`
   - GROUP BY user, ORDER BY summed XP DESC, LIMIT 25
   - Still need username/tier/level for each user (JOIN or separate lookup)

6. **User rank logic (when userId is provided):**
   - Count how many users have higher total_xp than this user: `rank = COUNT(*) + 1`
   - Get the user's own total_xp, level, tier, username
   - Compute gap_to_next: find the user ranked one position above and subtract XP values
   - If user is already #1, gap_to_next = 0
   - If user has no XP data, return userRank: null

7. **Error handling:**
   - Wrap in try/catch
   - On Supabase error: log with `[leaderboard]` prefix, return 503 `{ error: 'Leaderboard temporarily unavailable' }`
   - Never throw — match the never-fail pattern used throughout the backend

8. **Register in server.ts:**
   - Import: `import { router as leaderboardRouter } from './routes/leaderboard.js';`
   - Mount: `app.use('/api/leaderboard', leaderboardRouter);`
   - Place after the profile routes, before the server start

IMPORTANT: If Task 1 determines a SECURITY DEFINER function is needed, create it via `supabaseAdmin.rpc()` calling a raw SQL function. Use the Supabase SQL editor or create it via a migration script. Mirror the `get_ctc_xp_history` pattern exactly — SECURITY DEFINER, SET search_path = connect, returns JSON.
