<objective>
Enrich the XP award metadata stored in every transaction so the history panel can display meaningful row data.

Purpose: Phase 53's awardPlatformXp() only stores `metadata: { game_id: idempotencyKey }`. The history UI requires score, correct_answers, collection_slug, and is_duplicate per row. Without this fix, all history rows will show N/A for those fields. This plan patches the metadata at award time — the only place where these values are available. Existing history rows are unaffected (they keep N/A); all new games will have full data.

Output: Updated progressionService.ts with a metadata parameter, updated game.ts passing enriched metadata on every XP award.
</objective>

Add an optional `metadata` parameter to `awardPlatformXp()` that merges caller-supplied fields into the metadata object sent to the XP award API.

Current signature:
```
export async function awardPlatformXp(
  userId: string,
  amount: number,
  idempotencyKey: string
): Promise<XpAwardResult>
```

New signature:
```
export async function awardPlatformXp(
  userId: string,
  amount: number,
  idempotencyKey: string,
  metadata?: {
    score?: number;
    correctAnswers?: number;
    collectionSlug?: string;
    isDuplicate?: boolean;
  }
): Promise<XpAwardResult>
```

In the `body` of the fetch call, change:
```
metadata: { game_id: idempotencyKey },
```
to:
```
metadata: { game_id: idempotencyKey, ...metadata },
```

This preserves `game_id` as the first field and merges any additional fields the caller supplies. The parameter is optional so existing callers (none in this codebase other than game.ts) compile without changes if not yet updated.

Do NOT change withRetry, the return type, error handling, or any other logic in the function.

  <action>
In the `GET /results/:sessionId` handler, locate the existing call to `awardPlatformXp()`:

```typescript
session.xpResult = await awardPlatformXp(session.userId, xpAmount, idempotencyKey);
```

Replace it with:

```typescript
session.xpResult = await awardPlatformXp(session.userId, xpAmount, idempotencyKey, {
  score: results.totalScore,
  correctAnswers: results.totalCorrect,
  collectionSlug: session.collectionSlug ?? 'federal-civics',
  isDuplicate: false,  // placeholder — is_duplicate comes back in the response, not known at call time
});
```

Note on `isDuplicate`: The API returns `is_duplicate` in the RESPONSE, not before the call. Storing `false` at call time is intentional — the actual duplicate status is recorded in `session.xpResult.isDuplicate` when the response comes back. The history entry's `is_duplicate` in transaction metadata is necessarily the award-time value, so `false` is correct here. The display logic in the history panel should read `is_duplicate` from the transaction record itself (populated by the platform from the idempotency check), not from our metadata field.

Do NOT change anything else in the results handler — only the awardPlatformXp call site.
