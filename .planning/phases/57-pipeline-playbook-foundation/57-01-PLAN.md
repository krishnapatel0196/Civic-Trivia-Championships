This function implements the within-collection semantic dedup step:

1. Guard: if `!process.env.OPENAI_API_KEY`, print `"\n[Semantic Dedup] Skipping — OPENAI_API_KEY not set."` and return immediately.

2. Print `"\n[Step 5] Running within-collection semantic near-duplicate detection..."`.

3. Query the DB for all questions matching the collection prefix with status='draft' or 'active':
   ```ts
   const { db } = await import('../../db/index.js');
   const { questions } = await import('../../db/schema.js');
   const { sql } = await import('drizzle-orm');
   const prefixPattern = prefix + '-%';
   const rows = await db
     .select({ id: questions.id, externalId: questions.externalId, text: questions.text, options: questions.options, correctAnswer: questions.correctAnswer, qualityScore: questions.qualityScore })
     .from(questions)
     .where(sql`${questions.externalId} LIKE ${prefixPattern} AND ${questions.status} IN ('draft', 'active')`);
   ```

4. If `rows.length < 2`, print `"  [Semantic Dedup] Only ${rows.length} questions found — skipping (need at least 2)."` and return.

5. Print `"  Embedding ${rows.length} questions..."`.

6. Instantiate OpenAIEmbeddingService using the same cache dir as scan-duplicates.ts:
   ```ts
   const { OpenAIEmbeddingService } = await import('../../services/embeddings/OpenAIEmbeddingService.js');
   const { SemanticDupDetector } = await import('../../services/embeddings/SemanticDupDetector.js');
   const { ClusterBuilder } = await import('../../services/embeddings/ClusterBuilder.js');
   const { resolve } = await import('path');
   const cacheDir = resolve(process.cwd(), '..', '.embedding-cache');
   const embeddingService = new OpenAIEmbeddingService({ apiKey: process.env.OPENAI_API_KEY!, cacheDir });
   ```

7. Prepare texts using `SemanticDupDetector.prepareTextForEmbedding(q)`. Map rows into the `QuestionForDedup` shape:
   ```ts
   const questionsForDedup = rows.map(r => ({
     externalId: r.externalId,
     text: r.text,
     options: r.options as string[],
     correctAnswer: r.correctAnswer,
     qualityScore: r.qualityScore,
     collections: [collectionSlug],
   }));
   const textsToEmbed = questionsForDedup.map(q => ({
     id: q.externalId,
     text: SemanticDupDetector.prepareTextForEmbedding(q),
   }));
   ```

8. Call `embeddingService.embedBatch(textsToEmbed)` (no progress callback needed). Save cache: `embeddingService.saveCache()`.

9. Find pairs at 'near-duplicate' tier (threshold >0.85):
   ```ts
   const pairs = SemanticDupDetector.findAllPairs(questionsForDedup, embeddings, 'near-duplicate');
   ```

10. If `pairs.length === 0`, print `"  [Semantic Dedup] No near-duplicate pairs found. Collection looks clean."` and return.

11. Build clusters with a minimal tier map (within-collection, no hierarchy needed):
    ```ts
    const tierMap = new Map<string, string>();
    const clusterBuilder = new ClusterBuilder(tierMap as any);
    const questionMap = new Map(questionsForDedup.map(q => [q.externalId, q]));
    const clusters = clusterBuilder.buildClusters(pairs, questionMap as any);
    ```

12. For each cluster, determine the keep question (first in `cluster.recommendation.keep`) and archive the rest. For each archived question:
    - Update status to 'archived' in the DB:
      ```ts
      await db.update(questions)
        .set({ status: 'archived' })
        .where(sql`${questions.externalId} = ${archiveId}`);
      ```
    - Print: `"  [Semantic Dedup] Archived ${archiveId} (cluster ${cluster.clusterId}, score ${maxScore.toFixed(3)}, kept ${cluster.recommendation.keep})"`

13. Print final summary:
    ```
    "  [Semantic Dedup] Complete: ${clusters.length} duplicate cluster(s) found, ${totalArchived} question(s) archived."
    ```

Important constraints:
- Scope is within-collection ONLY (prefix-filtered query). Do NOT query all collections.
- Only archive questions in clusters where recommendation.archive has entries. Never archive the keep question.
- If DB update fails for a specific question, log the error and continue (don't abort the whole step).
- Import all embedding services via dynamic import (same pattern as the DB imports later in the file to avoid ESM issues).
