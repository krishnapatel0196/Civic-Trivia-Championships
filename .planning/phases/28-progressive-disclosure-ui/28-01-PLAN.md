  <action>
    **In `backend/src/services/feedbackService.ts`**, add a new exported function `updateFlagElaborations`:

    ```typescript
    export async function updateFlagElaborations(
      userId: number,
      sessionId: string,
      elaborations: Array<{
        questionId: string;  // externalId
        reasons: string[];
        elaborationText: string;
      }>
    ): Promise<number>
    ```

    Implementation:
    - Wrap all updates in `db.transaction()` for atomicity (all-or-nothing)
    - For each elaboration in the array:
      1. Look up question by `externalId` from `questions` table (same pattern as existing createFlag)
      2. Update the `questionFlags` row matching (userId, questionId, sessionId) — set `reasons` (jsonb array or null if empty) and `elaborationText` (trimmed string or null if empty)
      3. Track count of rows actually updated
    - Return total updatedCount
    - If any question not found, throw error (transaction rolls back all changes)
    - Use `eq` and `and` from drizzle-orm for WHERE clauses (same imports already in file)

    **In `backend/src/routes/feedback.ts`**, add a new PATCH route:

    ```
    PATCH /flags/batch
    ```

    - Apply `authenticateToken` middleware (same as existing routes)
    - NO rate limiter needed (elaboration is infrequent, one-time post-game action)
    - Validate with express-validator:
      - `body('sessionId').isString().notEmpty()`
      - `body('elaborations').isArray({ min: 1 })`
      - `body('elaborations.*.questionId').isString().notEmpty()`
      - `body('elaborations.*.reasons').isArray()`
      - `body('elaborations.*.reasons.*').isIn(['confusing-wording', 'outdated-info', 'wrong-answer', 'not-interesting'])`
      - `body('elaborations.*.elaborationText').isString().isLength({ max: 500 })`
    - On success: return `{ success: true, updatedCount: number }` with status 200
    - On validation error: return 400 with error details
    - On server error: return 500 with generic error message
    - Import `updateFlagElaborations` from feedbackService

    **Do NOT** modify existing POST /flag or DELETE /flag routes.
  </action>
