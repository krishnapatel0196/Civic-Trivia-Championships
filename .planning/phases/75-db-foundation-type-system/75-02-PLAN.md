**2a. Add InternationalLocaleConfig to `backend/src/scripts/content-generation/locale-configs/bloomington-in.ts`:**

Add the following interface export AFTER the existing `LocaleConfig` interface definition (around line 31, after the closing brace of LocaleConfig):

```typescript
/**
 * Extended config for International collections.
 * Required shape for any international locale config file (Phase 77+).
 * Extends LocaleConfig with RSS pipeline and pool management fields.
 */
export interface InternationalLocaleConfig extends LocaleConfig {
  /** RSS feed URLs for this collection (replaces sourceUrls for international) */
  rssFeeds: string[];
  /** Default confidence tier assigned to generated questions */
  confidenceTierDefault: 'high' | 'medium' | 'low';
  /** Trigger generation when active question count drops below this */
  poolFloor: number;
  /** Stop generating when active count reaches this target */
  poolTarget: number;
  /** Hard cap — archive oldest questions above this count before generating */
  poolCeiling: number;
}
```

This must be exported from `bloomington-in.ts` because that file is the canonical location for all locale config type interfaces. `replacementGenerator.ts` and `generate-locale-questions.ts` both import `LocaleConfig` from here. Keeping `InternationalLocaleConfig` alongside it prevents import drift in downstream phases.

**2b. Add international skip guard to `backend/src/cron/replacementGenerator.ts`:**

In the `generateReplacement` function, add a tier check BEFORE the existing `tryLoadLocaleConfig` call (currently around line 52). The guard must query the DB for the collection's tier and return early if it's international.

Add these imports at the top of the try block (after the null topic guard at line 47-49, before the `tryLoadLocaleConfig` call at line 52):

```typescript
    // Guard: skip replacement generation for international collections
    // International questions are managed by the RSS pipeline, not expiration replacement
    const { db: dbForTier } = await import('../db/index.js');
    const { collections } = await import('../db/schema.js');
    const { eq: eqForTier } = await import('drizzle-orm');
    const collectionRows = await dbForTier
      .select({ tier: collections.tier })
      .from(collections)
      .where(eqForTier(collections.id, collectionId))
      .limit(1);
    if (collectionRows[0]?.tier === 'international') {
      return { replaced: false, reason: 'international-collection-no-replacement' };
    }
```

Place this AFTER the `if (topic === null)` guard (line 47-49) and BEFORE the `const config = await tryLoadLocaleConfig(collectionSlug);` call (line 52).

Why here and not in expirationSweep.ts: The sweep correctly archives expired questions regardless of tier. Only the replacement generation must be skipped for international collections, which are managed by the RSS pipeline (Phase 77+). Placing the guard in replacementGenerator.ts makes it authoritative for all callers.
