  <action>
Create `backend/src/scripts/reactivate-archived-questions.ts` that does the following:

1. **Query archived questions per collection.** Use raw SQL similar to audit-questions.ts:
   ```sql
   SELECT q.id, q.external_id, q.text, q.options, q.correct_answer,
          q.explanation, q.difficulty, q.source, c.slug as collection_slug
   FROM civic_trivia.questions q
   JOIN civic_trivia.collection_questions cq ON q.id = cq.question_id
   JOIN civic_trivia.collections c ON cq.collection_id = c.id
   WHERE q.status = 'archived'
   ORDER BY c.slug, q.external_id
   ```

2. **Run each archived question through the quality rules engine.** Import `auditQuestion` from `../services/qualityRules/index.js`. Use `{ skipUrlCheck: true }` for speed (URL checks are fragile and not blocking for re-activation). Build a `QuestionInput` for each row:
   ```ts
   const input: QuestionInput = {
     externalId: row.external_id,
     text: row.text,
     options: row.options,
     correctAnswer: row.correct_answer,
     explanation: row.explanation,
     difficulty: row.difficulty,
     source: row.source,
   };
   ```

3. **Partition into pass/fail.** A question passes if `result.hasBlockingViolations === false`. Advisory-only violations are acceptable (same standard as active questions).

4. **Re-activate passing questions.** For each collection, UPDATE in a single batch:
   ```ts
   await db.update(questions)
     .set({ status: 'active', updatedAt: new Date() })
     .where(inArray(questions.externalId, passingExternalIds));
   ```

5. **Update quality scores for re-activated questions.** Set `qualityScore` from the audit result:
   ```ts
   for (const result of passingResults) {
     await db.update(questions)
       .set({ qualityScore: result.score })
       .where(eq(questions.externalId, result.question.externalId));
   }
   ```

6. **Print detailed per-collection summary.** For each of the 5 non-Federal collections, log:
   - Archived count (before)
   - Passing quality audit count
   - Failing quality audit count (with externalIds and violation reasons)
   - Re-activated count
   - New active total (query fresh count after update)
   - Remaining deficit to reach 90

7. **Print a final summary table** showing all 6 collections with their new active counts and whether each meets the 90 target.

8. **Support --dry-run flag.** When `process.argv.includes('--dry-run')`, do NOT execute the UPDATE statements. Just print what would be re-activated. This allows safe testing.

End the script with `process.exit(0)` in the success path and `process.exit(1)` in the error path. Call `pool.end()` in a finally block (same pattern as seed-community.ts).
