  <action>
Create `backend/src/cron/electionDetection.ts` with the following:

**Exports:**
- `CronRunSummary` interface: `{ timestamp: string; racesDetected: number; processed: number; failures: number; failureDetails: Array<{ raceId: number; seat: string; error: string }> }`
- `lastCronRun: CronRunSummary | null` â€” mutable module-level variable, initially null. Updated after each cron run. Consumed by the admin API endpoint (Plan 02) for the banner.
- `runElectionDetection(): Promise<void>` â€” the main cron function.

**runElectionDetection implementation:**
1. Record `startTime = Date.now()` and `now = new Date()`.
2. Compute `sixtyDaysFromNow = new Date(now.getTime() + 60 * 24 * 60 * 60 * 1000)`.
3. Query `electionRaces` where `questionsGenerated = false AND electionDate >= now AND electionDate <= sixtyDaysFromNow`. Use Drizzle `and()`, `eq()`, `gte()`, `lte()` operators. Import from `drizzle-orm`.
4. If no races found: log structured JSON (`level: 'info', job: 'election-detection', message: 'No upcoming elections requiring generation'`) and update `lastCronRun` with zeros. Return.
5. For each race, resolve the collection slug:
   - Query `collections` table: `SELECT slug FROM collections WHERE name = race.jurisdiction LIMIT 1`.
   - If no collection found: log as failure `"No collection found matching jurisdiction '{jurisdiction}'"` and add to failures array. Do NOT retry (a missing collection won't be fixed by retrying). Continue to next race.
6. For each race with a valid collection slug, attempt generation with retry (up to 3 attempts):
   - Call `generateElectionQuestions(race.id, collectionSlug, { force: false })`.
   - On success: increment `processed` counter, log success with raceId and seat.
   - On failure: if attempt < 3, retry immediately. If attempt === 3, add to `failures` array with `{ raceId: race.id, seat: race.seat, error: err.message }`.
   - IMPORTANT: If the error is `GenerationBlockedError`, do NOT retry â€” this means the race was already processed (race condition). Log it as skipped, not failed.
7. After all races processed, build `CronRunSummary` and assign to `lastCronRun`.
8. Log structured JSON summary: `{ level: 'info', job: 'election-detection', message: 'Election detection complete', racesDetected, processed, failures, failureDetails, durationMs, timestamp }`.

**Imports needed:**
- `{ db }` from `'../db/index.js'`
- `{ electionRaces, collections }` from `'../db/schema.js'`
- `{ and, eq, lte, gte }` from `'drizzle-orm'`
- `{ generateElectionQuestions, GenerationBlockedError }` from `'../services/generation/ElectionQuestionGenerator.js'`

**Structured logging:** Follow the exact same `console.log(JSON.stringify({...}))` pattern used in `expirationSweep.ts`. Use `level: 'info'` for normal operations, `level: 'warn'` for skipped races, `level: 'error'` for failures.

**Wrap the entire function body in try/catch** â€” if the top-level query fails, log the error and set `lastCronRun` to reflect the failure. Never let the cron crash the process.
