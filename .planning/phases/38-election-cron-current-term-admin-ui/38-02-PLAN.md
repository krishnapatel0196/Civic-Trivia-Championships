**Exports:**
- `FollowupBlockedError` class (extends Error) — thrown when `followupGenerated = TRUE`. Properties: `raceId: number`. Message: `"Current-term questions already generated for race {raceId}."`.
- `CurrentTermResult` interface: `{ questionsCreated: number; raceId: number; jurisdiction: string; collectionSlug: string }`.
- `generateCurrentTermQuestions(raceId: number, collectionSlug: string, opts: { winnerName: string; termEndDate: string }): Promise<CurrentTermResult>` — the main export.

**generateCurrentTermQuestions implementation:**
1. Load race from DB by `raceId`. Throw if not found.
2. Check `race.followupGenerated` — if TRUE, throw `FollowupBlockedError(raceId)`.
3. Resolve collection + topic using `resolveCollectionAndTopic(collectionSlug)` imported from `ElectionQuestionGenerator.ts`.
4. Compute `expiresAt = getEndOfDayUTC(opts.termEndDate, race.timezone)` — imported from `ElectionQuestionGenerator.ts`. This ensures current-term questions expire at end-of-day on the term end date in the jurisdiction's local timezone.
5. Build a current-term system prompt (see below).
6. Call Claude API using `client.messages.create()` with model `'claude-sonnet-4-6'`, max_tokens 4096, the system prompt, and user message: `"Generate 6 current-term civic trivia questions about ${opts.winnerName} as ${race.seat} in ${race.jurisdiction}."`.
7. Parse JSON response (same JSON extraction pattern as ElectionQuestionGenerator — regex match for `{...}`, parse, validate fields).
8. Insert questions with:
   - `externalId`: `elc-term-{raceId}-{seq}` format (e.g., `elc-term-5-001`). Use padded 3-digit seq.
   - `status: 'draft'`
   - `electionRaceId: raceId`
   - `expiresAt`: the computed value from step 4
   - `topicId`: from resolveCollectionAndTopic
   - `subcategory: 'current-term'`
   - Link to collection via `collectionQuestions` insert (same pattern as ElectionQuestionGenerator).
   - Use `onConflictDoNothing()` on both inserts.
9. Update `electionRaces` for this race: set `followupGenerated = true` AND `result = opts.winnerName`.
10. Return `CurrentTermResult`.

- Term End Date: ${opts.termEndDate}
- Election Type: ${race.electionType}

## Question Types

5. "Who won the [year] [seat] election in [jurisdiction]?" — election context (historical fact)
6. One additional office-specific question (salary, term length, how many members, notable powers, etc.)

## Rules
- All questions must be factual and non-partisan
- Do NOT compare candidates or express opinions
- Distractors should be plausible: real names of other officials, real dates, real government structures

${QUALITY_GUIDELINES}

Classification logic (strict priority — research confirmed):
1. **Awaiting Follow-up:** `election_date < now AND followup_generated = FALSE` (regardless of questions_generated)
2. **Active Elections:** `questions_generated = TRUE AND election_date >= now`
3. **Pending Generation:** `questions_generated = FALSE AND election_date >= now`
Races with `followup_generated = TRUE` are complete — excluded from all tabs.

For each race in the response, include a `questionCount` field: count of questions where `electionRaceId = race.id AND status IN ('active', 'draft')`.

Response shape: `{ activeElections: Race[], pendingGeneration: Race[], awaitingFollowup: Race[], cronLastRun: CronRunSummary | null }`.

**Endpoint 2: GET /election-races/:id/question-count**
Returns `{ activeCount: number, draftCount: number }` for a given race ID. Used by the re-generate confirmation modal to show "X active questions will be archived."

Query: `SELECT count(*) FROM questions WHERE election_race_id = :id AND status = 'active'` (and same for 'draft').

**Endpoint 3: POST /election-races/:id/enter-result**
Validates body with `enterResultSchema`. Calls `generateCurrentTermQuestions(raceId, collectionSlug, { winnerName, termEndDate })`.
- 200: returns `{ questionsCreated, raceId, jurisdiction, collectionSlug }`
- 400: validation error
- 404: race not found
- 409: `FollowupBlockedError` (already generated)
- 500: generation error (with `detail` field)

**Endpoint 4: POST /election-races/:id/regenerate**
This is the destructive re-generate flow. Steps:
1. Load race by ID. Return 404 if not found.
2. Resolve collection slug from `race.jurisdiction` by looking up `collections.name`. If no match, return 400: `"No collection found matching jurisdiction."`.
3. Archive active questions: `UPDATE questions SET status='archived', expiration_history = ... WHERE election_race_id = :id AND status = 'active'`. Use the same `expirationHistory` append pattern as `ElectionQuestionGenerator.ts`.
4. Delete draft questions: `DELETE FROM questions WHERE election_race_id = :id AND status = 'draft'`. Also delete corresponding `collection_questions` rows (use a subquery or fetch IDs first).
5. Reset `questionsGenerated = false` on the race.
6. Call `generateElectionQuestions(raceId, collectionSlug, { force: false })` — note: NOT `force: true` because we already manually archived/deleted. The race flag is reset to false so the normal generation path works.
7. Return `{ questionsCreated, archived, deleted, raceId, jurisdiction, collectionSlug }`.

**Endpoint 5: PUT /election-races/:id**
Update an existing race. Validates body with `updateRaceSchema`. Only updates fields present in the body. Return updated race.
- 200: `{ race: updated }`
- 400: validation error
- 404: race not found

Also add **DELETE /election-races/:id** — deletes a race record. Questions with `election_race_id` referencing this race will have it set to NULL (schema has `onDelete: 'set null'`). Return 200: `{ deleted: true }`.
