Create `backend/src/scripts/audit-collection-readiness.ts` — a standalone CLI script that performs a blocking pre-activation readiness audit for any collection.

**CLI interface:**
```
npx tsx src/scripts/audit-collection-readiness.ts --slug <slug> --prefix <prefix>
```

**Script must:**
1. Accept `--slug` and `--prefix` CLI args (same pattern as activate-collection.ts args parsing)
2. Verify the collection exists in DB and is currently inactive (warn but don't block if already active)
3. Count DRAFT questions matching `{prefix}-%` externalId pattern
4. Count ACTIVE questions matching `{prefix}-%` externalId pattern (important for Fremont — curation may have already run)
5. Count questions (draft + active) that have non-null `expiresAt` within 90 days of now (near-term expirations)
6. Calculate `netCount = (draftCount + activeCount) - expiringCount`
7. Print a full audit report: collection name, is_active status, draft count, active count, expiring count, net count, READY/NOT READY verdict
8. Exit code 0 if netCount >= 50 (READY), exit code 1 if netCount < 50 (BLOCKED)

**Critical rules:**
- Use Drizzle ORM imports from `../db/index.js` and `../db/schema.js` — these reference the `trivia` schema via `triviaSchema = pgSchema('trivia')`
- NEVER use `civic_trivia.` in any raw SQL — the schema is `trivia`
- Import: `import 'dotenv/config'` at top, then `import { db } from '../db/index.js'` and `import { questions, collections } from '../db/schema.js'`
- Import `eq, sql` from `drizzle-orm`
