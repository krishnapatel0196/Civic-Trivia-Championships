  <how-to-verify>
Review the generated Cambridge questions for quality and accuracy. Focus on the known pitfall areas from the research:

1. **Factual accuracy checks (blocking):**
   - No question asserts Cambridge was "the first US city" to pass a living wage ordinance (Baltimore was first, 1994; Cambridge passed theirs in May 1999)
   - No question describes Plan E as Cambridge "abolishing PR in 1938" — the 1938 vote REJECTED introducing PR; it was approved in 1940, first election 1941
   - No question describes Cambridge City Hall as "neoclassical" — it is Richardsonian Romanesque (1888)
   - No question implies the Cambridge Mayor has chief executive powers — the City Manager (appointed) is the chief executive

2. **Content balance checks:**
   - Harvard/MIT are not primary subjects of more than 2-3 questions
   - The collection represents more than just the university-adjacent population — look for at least some questions about East Cambridge, Cambridgeport, Portuguese/Azorean community, CRLS, or housing policy
   - Questions feel like "dinner party" civic tidbits, not address/phone number admin trivia

3. **Difficulty calibration:**
   - The mix skews toward Easy questions (appropriate for Easy Steps design)
   - No question has an address or phone number as its answer

4. **Expiring questions:**
   - Questions about specific current officeholders should have expiresAt set (not null)
   - Structural questions ("How are Cambridge City Councillors elected?") should have expiresAt null

**To review questions:** Use the admin UI or query the DB for questions with externalIdPrefix like 'cam%' and status = 'draft'.

**If you find issues:**
- Archive individual bad questions via the admin UI or the archive script
- If a specific factual topic has systemic errors (e.g., all PR history questions are wrong), archive that cluster and describe what needs fixing so Claude can adjust the voice guidance and regenerate
