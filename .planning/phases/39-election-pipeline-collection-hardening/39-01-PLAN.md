<objective>
Close three operational risks in the election pipeline identified by the v1.7 milestone audit:

1. **Admin collections endpoint** -- ElectionsPage jurisdiction dropdown currently fetches from `GET /api/game/collections` which applies a 50-question floor, hiding new/sparse collections. Add `GET /api/admin/collections` that returns all collections unfiltered.

2. **Jurisdiction validation** -- `POST /election-races` accepts any jurisdiction string with no DB validation. Add a lookup against `collections.name` to return 400 if the jurisdiction does not match any collection.

3. **Regenerate collection override** -- `POST /election-races/:id/regenerate` resolves the collection slug exclusively via `collections.name = race.jurisdiction`. Add an optional `collectionSlug` body parameter that overrides the name-based lookup when provided.

Purpose: These three surgical fixes ensure the election pipeline cannot silently accept bad data and gives admins an escape hatch when collections are renamed.
</objective>
