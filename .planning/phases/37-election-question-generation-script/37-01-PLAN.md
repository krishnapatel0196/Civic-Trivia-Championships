  <action>
Create `backend/src/services/generation/ElectionQuestionGenerator.ts` with the following components:

**1. getEndOfDayUTC(localDateStr: string, tz: string): Date**
Timezone-aware end-of-day computation using built-in `Intl.DateTimeFormat`. Copy the verified implementation from RESEARCH.md Pattern 8. This computes the UTC equivalent of 23:59:59.999 on the given local date in the given IANA timezone. No external date library — use only Node.js built-ins.

**2. getQuestionTarget(race: ElectionRace): number**
Returns question count by seat type: 15 for mayor (seat.toLowerCase().includes('mayor')), 10 for council (seat.toLowerCase().includes('council')), 8 for everything else. Inspect `race.seat`, not `race.electionType`.

**3. buildElectionSystemPrompt(race: ElectionRace, questionTarget: number): string**
Build a system prompt following the structure in RESEARCH.md Pattern 10 (Election Question System Prompt Structure). Must include:
- Race details block: seat, electionType, jurisdiction, electionDate (formatted YYYY-MM-DD), candidate names
- Question types: seat-civics + candidate-about (factual only, no comparisons)
- MCQ construction guidance conditional on candidate count (ELEC-09):
  - 2 candidates: avoid "which candidate?" questions with only 2 options; use real government entities as distractors
  - 5+ candidates: focus one question on one candidate; distractors can be other real candidates
  - 3-4 candidates: standard mix
- Output format: JSON with questions array, each having text, options (4), correctAnswer (0-based index), explanation (must include "According to"), difficulty (easy/medium/hard), topicCategory ("election-race"), source ({name, url})
- Embed QUALITY_GUIDELINES from `../scripts/content-generation/prompts/quality-guidelines.ts` (import the constant)

**4. buildExternalId(raceId: number, seq: number, force: boolean): string**
Normal: `elc-${raceId}-${String(seq).padStart(3, '0')}` (e.g., `elc-1-003`)
Force: `elc-${raceId}-${Date.now().toString(36)}-${String(seq).padStart(3, '0')}` to avoid ON CONFLICT collisions with archived questions.

**5. GenerationBlockedError class**
Custom error with `existingCount: number` and `generatedAt: string | null` properties. Thrown when `questionsGenerated = TRUE` and `force` is false.

**6. resolveCollectionAndTopic(collectionSlug: string): Promise<{ collectionId: number; topicId: number }>**
- Look up collection by slug. Fail fast if not found.
- Look up the `elections-voting` topic linked to that collection via `collectionTopics` JOIN `topics` WHERE `topics.slug = 'elections-voting'`.
- If `elections-voting` topic does not exist for this collection: create it (insert into topics, then link via collectionTopics). Use name "Elections & Voting", slug "elections-voting", description "Questions about elections and voting in this jurisdiction".
- Return both IDs.

**7. generateElectionQuestions(raceId: number, collectionSlug: string, opts: { force?: boolean; dryRun?: boolean }): Promise<GenerationResult>**

The main export. Steps:
1. Load race from DB by ID. Fail if not found.
2. Check `questionsGenerated` flag. If TRUE and not force, throw `GenerationBlockedError` with count of existing questions for this race and createdAt of the latest one.
3. If force: archive existing questions (set status='archived', append to expirationHistory), then reset `questionsGenerated = false`.
4. Resolve collection + topic via `resolveCollectionAndTopic(collectionSlug)`.
5. Compute expiresAt: extract local date string from race.electionDate using `Intl.DateTimeFormat('en-CA', { timeZone: race.timezone })`, then call `getEndOfDayUTC(localDateStr, race.timezone)`.
6. Determine question target count via `getQuestionTarget(race)`.
7. Build system prompt via `buildElectionSystemPrompt(race, questionTarget)`.
8. Build user message: "Generate exactly {questionTarget} multiple-choice civic trivia questions about the {race.seat} race in {race.jurisdiction}."
9. Call Claude API: use Anthropic client (import from `../../scripts/content-generation/anthropic-client.js`), model `'claude-sonnet-4-6'` (NOT the MODEL constant which is outdated at claude-sonnet-4-5), max_tokens 8192, system prompt separate from user message.
10. Parse response: extract JSON from response text using regex `\{[\s\S]*\}`, parse with JSON.parse. Validate structure loosely (check questions array exists, each has required fields). Do NOT use the existing QuestionSchema/BatchSchema (the externalId regex won't match election format and the min batch size is wrong for elections).
11. If dryRun: log questions to console and return result with questionsCreated = parsed count, archived = 0.
12. For each parsed question, insert into DB:
    - `externalId`: via `buildExternalId(raceId, seq, force)` where seq starts at 1
    - `text`, `options`, `correctAnswer`, `explanation`, `difficulty`: from parsed question
    - `topicId`: from resolved topic
    - `subcategory`: 'election-race'
    - `source`: from parsed question
    - `learningContent`: null
    - `expiresAt`: the computed end-of-day UTC date
    - `status`: 'draft'
    - `expirationHistory`: []
    - `electionRaceId`: raceId
    - Use `db.insert(questions).values({...}).onConflictDoNothing().returning({ id: questions.id })`
13. For each successfully inserted question: also insert into `collectionQuestions` with the resolved collectionId.
14. Set `questionsGenerated = true` on the race: `db.update(electionRaces).set({ questionsGenerated: true }).where(eq(electionRaces.id, raceId))`.
15. Return `GenerationResult` with questionsCreated, archived count, raceId, jurisdiction, collectionSlug.

**Interface exports:**
```typescript
export interface GenerationResult {
  questionsCreated: number;
  archived: number;
  raceId: number;
  jurisdiction: string;
  collectionSlug: string;
}
```
