**1. Implement undo toast in FlagDetailPanel.tsx:**

Add a simple toast component and useUndoToast hook INSIDE FlagDetailPanel.tsx (no separate file needed — it's ~50 lines).

```typescript
interface ToastState {
  message: string;
  undoCallback: (() => void) | null;
  visible: boolean;
}
```

- `showToast(message, undoCallback)`: sets visible=true, starts 5-second timeout, clears previous timeout
- `hideToast()`: sets visible=false, clears timeout
- Toast renders as fixed bottom-4 right-4 div: `bg-gray-900 text-white px-4 py-3 rounded-lg shadow-lg flex items-center gap-3 z-50`
- Shows message text + "Undo" button (text-amber-400 hover:text-amber-300 font-medium) + close X button
- When Undo clicked: call undoCallback, hide toast
- When timeout expires: toast auto-hides (action is already committed at this point)

**2. Implement action handlers in FlagDetailPanel.tsx:**

**Archive handler:**
- Optimistically call `onQuestionArchived(questionId)` (removes from parent table immediately)
- Close the detail panel
- Show undo toast: "Question archived"
- Call API: `PATCH /api/admin/flags/${questionId}/archive`
- Undo callback: call `PATCH /api/admin/flags/${questionId}/restore` to reverse, then tell parent to re-fetch by calling a new `onRefreshNeeded` prop (add this prop to the interface)
- On API error: show error, call onRefreshNeeded to restore correct state
- Track pending state to disable buttons while action in flight

**Dismiss handler:**
- Show a brief confirm: `window.confirm('Dismiss all flags for this question? This will remove all player feedback.')` — this is the ONE action that deserves confirmation since it deletes data permanently
- Optimistically call `onFlagsDismissed(questionId)` (removes from parent table)
- Close the detail panel
- Show undo toast... actually dismiss is NOT undoable (hard delete). So NO undo toast — just confirm dialog.
- Call API: `POST /api/admin/flags/${questionId}/dismiss`
- On error: show error toast (no undo), call onRefreshNeeded

**Restore handler:**
- Optimistically call `onQuestionRestored(questionId)` (removes from archived tab)
- Close the detail panel
- Call API: `PATCH /api/admin/flags/${questionId}/restore`
- No undo toast needed (user can just archive again)
- On error: show error, call onRefreshNeeded
