**Create `backend/src/services/qualityRules/rules/address-phone.ts`:**

Follow the exact pattern from `partisan.ts`. Import types with `.js` extension (ESM requirement):
```typescript
import { RuleResult, QuestionInput, Violation } from '../types.js';
```

Define three regex patterns:
- US phone: `/(\+1[\s.-]?)?\(?\d{3}\)?[\s.-]?\d{3}[\s.-]?\d{4}/`
- UK phone: `/(\+44[\s.-]?)?\(?0\d{1,4}\)?[\s.-]?\d{3,4}[\s.-]?\d{4}/`
- Street address: `/\d{1,5}\s+[A-Za-z\s]{2,40}(?:Street|St|Avenue|Ave|Road|Rd|Boulevard|Blvd|Drive|Dr|Lane|Ln|Court|Ct|Way|Place|Pl|Circle|Cir)\.?\b/i`

Export function `checkAddressPhone(question: QuestionInput): RuleResult`.

CRITICAL: Scan ONLY `question.options` (the 4 answer choices). Do NOT scan `question.text` or `question.explanation`. Civic questions legitimately reference addresses in the question body (e.g., "The Supreme Court is located at 1 First Street NE"). The problem is answer CHOICES containing contact info.

For each option, test all three regexes. If any match, create an advisory violation:
```typescript
{
  rule: 'address-phone',
  severity: 'advisory',
  message: 'Answer option contains a phone number or street address',
  evidence: // join all matched patterns for this option
}
```

Return `{ passed: violations.length === 0, violations }`.
