Table columns:
- `id: serial('id').primaryKey()`
- `seat: text('seat').notNull()` — e.g., "Mayor of Bloomington"
- `electionType: text('election_type').notNull()` — values: primary, general, runoff, by-election
- `electionDate: timestamp('election_date', { withTimezone: true }).notNull()`
- `timezone: text('timezone').notNull()` — IANA format, e.g., "America/Indiana/Indianapolis"
- `jurisdiction: text('jurisdiction').notNull()` — e.g., "Bloomington, IN"
- `candidates: jsonb('candidates').$type<Array<{name: string; party: string; incumbent: boolean}>>().notNull().default([])`
- `questionsGenerated: boolean('questions_generated').notNull().default(false)`
- `followupGenerated: boolean('followup_generated').notNull().default(false)`
- `result: text('result')` — nullable, filled after election resolves
- `createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull()`

No indexes needed — table will have very few rows in v1.7.

Add type exports after the table definition:
```typescript
export type ElectionRace = typeof electionRaces.$inferSelect;
export type NewElectionRace = typeof electionRaces.$inferInsert;
```

Then add `electionRaceId` to the existing `questions` table definition (add it after `flagCount`, before the closing of the table definition):
```typescript
electionRaceId: integer('election_race_id')
  .references(() => electionRaces.id, { onDelete: 'set null' }),
```

This column is NULLABLE (no `.notNull()`) — most questions have no election race association. `ON DELETE SET NULL` means deleting a race preserves the questions.

After editing schema.ts, run `npm run db:migrate` from the `backend/` directory. This runs `drizzle-kit push` which applies the schema changes directly to the database. Do NOT run `npm run db:generate` — this project does not use generate+migrate.

**Backend — admin.ts:**

Add two routes to `backend/src/routes/admin.ts`. Import `electionRaces` from `../db/schema.js` (add to existing import). Import `desc` from `drizzle-orm` if not already imported.

1. `GET /election-races` — List all election races ordered by createdAt desc:
```typescript
router.get('/election-races', async (req: Request, res: Response) => {
  try {
    const races = await db.select().from(electionRaces).orderBy(desc(electionRaces.createdAt));
    res.json({ races });
  } catch (error) {
    console.error('Failed to fetch election races:', error);
    res.status(500).json({ error: 'Failed to fetch election races' });
  }
});
```

2. `POST /election-races` — Create a new election race. Validate with Zod (already imported in admin.ts):
```typescript
const createElectionRaceSchema = z.object({
  seat: z.string().min(1).max(200),
  electionType: z.enum(['primary', 'general', 'runoff', 'by-election']),
  electionDate: z.string().datetime({ offset: true }),
  timezone: z.string().min(1),
  jurisdiction: z.string().min(1).max(200),
  candidates: z.array(z.object({
    name: z.string().min(1),
    party: z.string().min(1),
    incumbent: z.boolean(),
  })).default([]),
});
```
Parse request body with `.safeParse()`, return 400 on validation failure. Insert into `electionRaces` table using `db.insert(electionRaces).values({...}).returning()`. Return 201 with the created race object.
