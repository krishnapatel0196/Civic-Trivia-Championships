  Create a new file `backend/src/scripts/repair-broken-links.ts` with the following structure:

  **Imports:**
  - `import '../env.js'` (loads dotenv)
  - `import { db } from '../db/index.js'`
  - `import { questions } from '../db/schema.js'`
  - `import { eq, and, isNotNull, sql } from 'drizzle-orm'`
  - `import { client, MODEL } from './content-generation/anthropic-client.js'`
  - `import pLimit from 'p-limit'`

  **CLI arguments parsing:**
  - `--scan` : Scan for broken links and report (default behavior)
  - `--batch N` : Process N questions at a time (default: 20)
  - `--apply` : Actually apply the repairs (without this, dry-run only)
  - `--concurrency N` : URL validation concurrency (default: 5)

  **URL Validation function (`validateUrl`):**
  - Use native fetch with method: 'HEAD', redirect: 'follow'
  - AbortSignal.timeout(5000) for 5-second timeout
  - User-Agent: 'CivicTriviaBot/1.0 (link validation)'
  - Return { url, isValid, statusCode, error? }
  - Wrap in try/catch, handle TimeoutError gracefully
  - If HEAD returns 405 Method Not Allowed, fall back to GET with same timeout

  **AI URL Discovery function (`findSourceUrls`):**
  - Use existing client and MODEL from anthropic-client
  - System prompt: "You are a civic education content curator. Find 1-3 authoritative source URLs for the given civic trivia question. ONLY suggest URLs from .gov, .edu, or established civic education organizations (e.g., archives.gov, constitutioncenter.org, uscis.gov). Respond with valid JSON."
  - Input: question text + correct answer text
  - Response schema: { urls: [{ url, source_name, relevance_score (1-10) }], confidence: 'high'|'medium'|'low' }
  - Parse JSON from response text, validate structure with basic checks
  - On parse error, return empty urls array with 'low' confidence
  - Use temperature: 0.3 for factual accuracy

  **Scan function (`scanBrokenLinks`):**
  - Query all active questions that have a non-null source URL
  - Use `sql` to filter where source->>'url' is not null and not empty
  - Batch validate current URLs using pLimit concurrency
  - Report: total scanned, broken count, valid count
  - Return array of questions with broken links

  **Repair function (`repairBrokenLinks`):**
  - For each broken question (in batches):
    1. Call findSourceUrls with question text and correct answer
    2. Validate all AI-suggested URLs via HTTP HEAD
    3. Pick best: highest relevance_score among valid URLs
    4. If best found and --apply: update question source to { name: suggestion.source_name, url: suggestion.url }
    5. If no valid suggestion and --apply: set source to { name: question.source.name, url: null }
    6. Log each decision: question ID, old URL, new URL or null, AI confidence
  - Add a 1-second delay between AI calls to avoid rate limiting
  - Print summary: total processed, repaired count, null-set count, failed count

  **Main function:**
  - Parse CLI args
  - Run scan first, report findings
  - If --apply, run repair and commit changes
  - If not --apply, show what would be changed (dry-run report)
  - Exit with code 0 on success, 1 on error
