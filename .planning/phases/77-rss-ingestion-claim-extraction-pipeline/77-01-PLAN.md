**File 1: `rss-ingestor.ts`**

This service handles RSS feed fetching, article text extraction, and the 300-word gate. It exports functions consumed by Plan 77-02's claim-extractor and question-generator.

Key types to export:
```typescript
export interface ParsedArticle {
  title: string;
  url: string;           // item.link
  pubDate: Date;         // parsed from item.pubDate or item.isoDate
  feedName: string;      // human-readable feed name (e.g., "BBC World")
  feedUrl: string;       // the RSS feed URL
  bodyText: string;      // extracted article text (passed 300-word gate)
  guid: string;          // item.guid or item.link as fallback
}

export interface FeedResult {
  feedUrl: string;
  feedName: string;
  articles: ParsedArticle[];
  error?: string;        // populated if feed fetch failed
}
```

**Feed configuration** — define as a const array (not hardcoded in function body). Phase 78 may move this to DB, but for now a typed constant is correct:
```typescript
export const INTERNATIONAL_FEEDS = [
  { name: 'BBC World', url: 'https://feeds.bbci.co.uk/news/world/rss.xml' },
  { name: 'NPR', url: 'https://feeds.npr.org/1004/rss.xml' },
  { name: 'The Guardian', url: 'https://www.theguardian.com/world/rss' },
  { name: 'DW', url: 'https://rss.dw.com/rdf/rss-en-world' },
] as const;
```

**`fetchAllFeeds(feedUrls?: typeof INTERNATIONAL_FEEDS)`** — main exported function:
- Uses `rss-parser` with `content:encoded` custom field support (see Pattern 2 in RESEARCH.md)
- Wraps each feed in its own try/catch — a single bad feed returns `{ feedUrl, feedName, articles: [], error }`, never throws
- For each RSS item: call `extractArticleBody()` to get full text
- Apply 300-word gate: `wordCount(bodyText) < 300` => skip and log `[{feedName}] Skipped (${wordCount} words): {title}`
- Parse pubDate: use `new Date(item.isoDate || item.pubDate)` — rss-parser normalizes to isoDate. If neither exists or is invalid, skip the article.
- Log per-feed summary: `[{feedName}] {N} articles fetched, {M} passed 300-word gate, {K} skipped`
- Return `FeedResult[]`

**`extractArticleBody(url: string)`** — exported for testing:
- Use `p-limit` to cap concurrent HTTP fetches at 5 (import from existing package)
- Fetch the URL with `User-Agent: CivicTriviaBot/1.0` and `AbortSignal.timeout(15000)`
- Parse HTML with cheerio: try selectors in order: `'article'`, `'main'`, `'[class*="article"]'`, `'[class*="content"]'`, `'body'`
- For each selector, get `.text()`, collapse whitespace, check if length > 200 chars
- Return the first match that exceeds 200 chars, or null
- On fetch error (403, timeout, etc.): return null (caller handles gracefully)

**`wordCount(text: string)`** — simple utility:
```typescript
export function wordCount(text: string): number {
  return text.split(/\s+/).filter(Boolean).length;
}
```

**RSS body-first strategy for article text resolution:**
1. ALWAYS attempt HTTP fetch for full body (BBC World has NO content:encoded)
2. If HTTP fetch returns null or < 300 words, try `content:encoded` from RSS (strip HTML with cheerio)
3. If still < 300 words, skip the article
This means the extractArticleBody call happens for every article, not just as a fallback.
