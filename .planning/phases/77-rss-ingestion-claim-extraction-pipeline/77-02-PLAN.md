Create the ClaimExtractor service that handles story deduplication and claim extraction.

**Key types to export:**
```typescript
export interface StoryCluster {
  articles: ParsedArticle[];  // 2+ articles about the same event
  sharedEntities: string[];   // entities that caused the clustering
  representativeTitle: string; // title from the article with most body text
}

export interface ClaimResult {
  claim: string;              // the single best verifiable claim
  factSnapshot: string;       // fact_snapshot string for DB
  confidenceTier: 'high' | 'medium' | 'low';
  sourceArticles: ParsedArticle[]; // articles in the cluster
}
```

**Part A: Named-Entity Extraction + Clustering**

`extractEntities(text: string): Set<string>` — internal helper:
- Use `compromise` library: `import nlp from 'compromise';`
- Extract people (`.people()`), places (`.places()`), organizations (`.organizations()`)
- Normalize: lowercase, trim
- Filter: minimum 3 characters (avoids noise like "US", "UN" — but keep "iran", "nato")
- Return Set<string>

`clusterArticles(articles: ParsedArticle[]): StoryCluster[]` — exported:
- For each article, extract entities from bodyText
- Cluster by: articles within 24-hour pubDate window of each other AND sharing 2+ named entities
- 24-hour window is article-to-article: `Math.abs(a.pubDate.getTime() - b.pubDate.getTime()) <= 24 * 60 * 60 * 1000`
- Use union-find or greedy clustering: iterate article pairs, merge into clusters when both conditions met
- **Filter: clusters with fewer than 2 articles are dropped** (2+ corroborating sources required)
- Log dropped single-source stories: `[Dedup] Skipped single-source story: "{title}" ({feedName})`
- Log formed clusters: `[Dedup] Cluster: "{representativeTitle}" — {N} articles from {feedNames}`
- The `representativeTitle` is the title from the article with the longest bodyText
- The `sharedEntities` field records which entities caused the match

**Clustering algorithm (greedy, simple):**
1. Build array of `{ article, entities: Set<string> }`
2. For each pair (i, j) where i < j:
   - Check 24-hour window AND 2+ shared entities
   - If both conditions met, union them into same cluster
3. Collect clusters, filter to size >= 2
4. For each cluster, pick representative title from longest-body article
