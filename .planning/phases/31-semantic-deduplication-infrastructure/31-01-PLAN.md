    **OpenAIEmbeddingService.ts:**

    Create a class `OpenAIEmbeddingService` that wraps the OpenAI embeddings API with caching and rate limiting.

    Constructor accepts `{ apiKey: string, cacheDir?: string }`. Defaults cacheDir to `backend/.embedding-cache`.
    - Create OpenAI client with `maxRetries: 3`, `timeout: 120000`
    - Create p-limit limiter with concurrency 10
    - Use model `text-embedding-3-small` (1536 dimensions)

    Cache strategy:
    - On construction, load cache from `{cacheDir}/embeddings.json` if it exists
    - Cache is a `Map<string, number[]>` keyed by normalized text (lowercase, trimmed, whitespace-collapsed)
    - `saveCache()` method writes cache to disk as JSON object (Object.fromEntries)
    - `getCacheStats()` returns `{ hits: number, misses: number, total: number }`
    - Create cache directory if it doesn't exist (use `mkdirSync` with `recursive: true`)

    Methods:
    - `async embed(text: string): Promise<number[]>` — Normalize text, check cache, call API if miss, store in cache, return 1536-dim vector. Wraps API call in p-limit limiter.
    - `async embedBatch(items: Array<{ id: string, text: string }>, onProgress?: (done: number, total: number) => void): Promise<Map<string, number[]>>` — Embed multiple texts, returns Map of id -> embedding. Calls `embed()` for each (p-limit handles concurrency). Reports progress via callback. Continues on individual failures (logs error, skips item).
    - `saveCache(): void` — Persist cache to disk
    - `getCacheStats(): { hits: number, misses: number, total: number }`

    Follow the pattern from `anthropic-client.ts` — simple, focused, SDK auto-configured.
    Use `import OpenAI from 'openai'` (ESM default import).
    Use `import pLimit from 'p-limit'` (ESM default import).
    Use `import { readFileSync, writeFileSync, existsSync, mkdirSync } from 'fs'`.

    **SemanticDupDetector.ts:**

    Create a class `SemanticDupDetector` with static methods for similarity detection.

    Import types from `./types.js`.

    Static methods:
    - `cosineSimilarity(vecA: number[], vecB: number[]): number` — Standard cosine similarity. Throw if dimensions don't match. Return value between -1 and 1 (embeddings typically 0-1).
    - `getSimilarityTier(score: number): SimilarityTier` — Returns 'exact' if >0.95, 'near-duplicate' if >0.85, 'possible' if >0.75, else 'unrelated'. Thresholds as named constants at top of file for easy tuning.
    - `normalizedTextSimilarity(textA: string, textB: string): number` — Fast text comparison: lowercase, strip punctuation, collapse whitespace, then check exact match (returns 1.0) or no match (returns 0.0). This is the "fast pass" for catching copy-paste duplicates before expensive embedding calls.
    - `prepareTextForEmbedding(question: QuestionForDedup): string` — Concatenates question text + all answer options into a single string for embedding. Format: `"{text}\nA: {options[0]}\nB: {options[1]}\nC: {options[2]}\nD: {options[3]}"`. This ensures questions with same text but different answers are distinguished.
    - `findAllPairs(questions: QuestionForDedup[], embeddings: Map<string, number[]>, minTier?: SimilarityTier): SimilarityResult[]` — Compare all N*(N-1)/2 pairs. First check normalizedTextSimilarity for fast exact matches. For remaining pairs, use cosine similarity on embeddings. Return only pairs meeting minTier threshold (default: 'possible'). Sort results by score descending.

    **ClusterBuilder.ts:**

    Create a class `ClusterBuilder` implementing union-find with path compression for clustering duplicate questions.

    Import types from `./types.js`.

    Private fields:
    - `parent: Map<string, string>` — union-find parent pointers
    - `rank: Map<string, number>` — union by rank for balanced trees

    Methods:
    - `find(id: string): string` — Find root with path compression
    - `union(id1: string, id2: string): void` — Union by rank
    - `buildClusters(pairs: SimilarityResult[], questions: Map<string, QuestionForDedup>): DuplicateCluster[]` — Main entry point:
      1. Union all pairs
      2. Group by root to form clusters
      3. Filter to clusters with 2+ members
      4. For each cluster, determine tier (highest similarity among pairs in cluster)
      5. For each cluster, compute recommendation using `recommendKeep()`
      6. Assign clusterId as "cluster-001", "cluster-002", etc.
      7. Sort clusters by tier severity (exact first, then near-duplicate, then possible)

    - `private recommendKeep(cluster questions: QuestionForDedup[]): ClusterRecommendation` — Determine which question to keep using cross-collection policy:
      1. If questions span different collection tiers: keep the one in the highest tier (Federal > State > City). Use `COLLECTION_HIERARCHY` and `TIER_RANK` from types.
      2. If same collection tier: keep the one with the highest qualityScore (null treated as 0).
      3. If tied on quality: keep the one with the lowest externalId (alphabetical stability).
      4. Archive all others.
      5. Reason string explains the decision (e.g., "Kept fed-q042 (Federal tier) over bloom-q015 (City tier)" or "Kept cal-q023 (qualityScore: 85) over cal-q067 (qualityScore: 72)")

    All three files must use `.js` extensions in import paths (ESM module resolution per tsconfig).
