    Create `backend/src/scripts/scan-duplicates.ts` following the pattern from `audit-questions.ts`.

    **Imports:**
    - `'../env.js'` (dotenv loading, same as audit-questions)
    - `{ db }` from `'../db/index.js'`
    - `{ sql }` from `'drizzle-orm'`
    - `{ OpenAIEmbeddingService }` from `'../services/embeddings/OpenAIEmbeddingService.js'`
    - `{ SemanticDupDetector }` from `'../services/embeddings/SemanticDupDetector.js'`
    - `{ ClusterBuilder }` from `'../services/embeddings/ClusterBuilder.js'`
    - Types from `'../services/embeddings/types.js'`
    - `{ writeFileSync, mkdirSync, existsSync }` from `'fs'`
    - `{ resolve }` from `'path'`

    **CLI flags (parseFlags function):**
    Follow audit-questions.ts pattern with `process.argv.slice(2)`.
    - `--dry-run`: Show what would happen, no database modifications (this is the default behavior for Phase 31 — the tool always generates reports, never modifies data. Flag is for explicit user clarity and sets messaging tone.)
    - `--collections=name1,name2`: Filter to specific collections (default: all)
    - `--skip-cache`: Ignore existing embedding cache, re-embed everything
    - `--min-tier=exact|near-duplicate|possible`: Minimum similarity tier to include in report (default: 'possible')

    **fetchQuestions function:**
    Query database for all active questions with collection memberships. Use raw SQL (same pattern as audit-questions.ts):
    ```sql
    SELECT
      q.external_id as "externalId",
      q.text,
      q.options,
      q.correct_answer as "correctAnswer",
      q.quality_score as "qualityScore",
      array_agg(c.name ORDER BY c.name) as "collections"
    FROM civic_trivia.questions q
    JOIN civic_trivia.collection_questions cq ON q.id = cq.question_id
    JOIN civic_trivia.collections c ON cq.collection_id = c.id
    WHERE q.status = 'active'
    GROUP BY q.id, q.external_id, q.text, q.options, q.correct_answer, q.quality_score
    ORDER BY q.external_id
    ```
    If `--collections` flag is set, add `AND c.name = ANY($1)` filter (use parameterized query via sql template).
    Map results to `QuestionForDedup[]`.

    **main function flow:**
    1. Parse flags, print banner:
       ```
       === Semantic Duplicate Scanner ===
       Mode: DRY RUN (report only, no changes)
       Collections: all (or filtered list)
       Minimum tier: possible
       ```

    2. Fetch questions from database. Print count: "Fetched {N} active questions across {M} collections"

    3. Validate OPENAI_API_KEY exists in environment. If missing, print error and exit(1):
       "Error: OPENAI_API_KEY not set. Add it to backend/.env"

    4. Create OpenAIEmbeddingService instance (pass `process.env.OPENAI_API_KEY`). If --skip-cache, pass a temp cache dir.

    5. Prepare texts for embedding using `SemanticDupDetector.prepareTextForEmbedding()` for each question.

    6. Embed all question texts via `embeddingService.embedBatch()`. Show progress:
       "Embedding questions... {done}/{total} ({cacheHits} cache hits)"

    7. Save embedding cache: `embeddingService.saveCache()`

    8. Find all similar pairs using `SemanticDupDetector.findAllPairs()` with the specified minTier.
       Print: "Found {N} similar pairs above {minTier} threshold"

    9. Build clusters using `ClusterBuilder.buildClusters()`.
       Print: "Grouped into {N} duplicate clusters"

    10. Generate reports:
        - Create output dir: `.planning/dedup-reports/` (relative to project root, use `resolve()` to find project root from script location)
        - Timestamp for filenames: `YYYY-MM-DD-HHmmss` format
        - JSON report: `.planning/dedup-reports/duplicates-{timestamp}.json`
        - Markdown report: `.planning/dedup-reports/duplicates-{timestamp}.md`

    11. Print summary to console:
        ```
        === Scan Complete ===

        Questions scanned: {N}
        Collections covered: {list}
        Similar pairs found: {N}
        Duplicate clusters: {N}
          - Exact (>0.95): {N} clusters
          - Near-duplicate (>0.85): {N} clusters
          - Possible (>0.75): {N} clusters

        Embedding cache: {hits} hits, {misses} new embeddings

        Reports saved:
          JSON: .planning/dedup-reports/duplicates-{timestamp}.json
          Markdown: .planning/dedup-reports/duplicates-{timestamp}.md

        DRY RUN: No changes made. Review reports before proceeding.
        ```

    **generateJsonReport function:**
    Write the DuplicateCluster[] array as formatted JSON (2-space indent).

    **generateMarkdownReport function:**
    Generate human-readable report following audit-questions.ts markdown pattern:

    ```markdown
    # Duplicate Detection Report

    **Generated:** {ISO timestamp}
    **Questions Scanned:** {N}
    **Collections:** {list}
    **Duplicate Clusters Found:** {N}

    ## Summary

    | Tier | Clusters | Questions Affected |
    |------|----------|--------------------|
    | Exact (>0.95) | {N} | {N} |
    | Near-duplicate (>0.85) | {N} | {N} |
    | Possible (>0.75) | {N} | {N} |

    ## Clusters

    ### Cluster 1 — Exact Match (score: 0.97)

    **Recommendation:** Keep `fed-q042` (Federal tier), archive `bloom-q015` (City tier)
    **Reason:** Kept in broader collection per Federal > State > City hierarchy

    #### fed-q042 (Federal Civics)
    **Question:** {full text}
    **Answers:**
      [checkmark] 1. {correct answer}
        2. {wrong answer}
        3. {wrong answer}
        4. {wrong answer}
    **Quality Score:** 85

    #### bloom-q015 (Bloomington, IN)
    **Question:** {full text}
    **Answers:**
    ... (same format)

    **Pairwise Similarities:**
    | Pair | Score | Tier |
    |------|-------|------|
    | fed-q042 vs bloom-q015 | 0.97 | exact |
    ```

    For each question in a cluster, show:
    - externalId and collection(s)
    - Full question text
    - All 4 answer options with checkmark on correct answer
    - Quality score (or "N/A" if null)

    **Error handling:**
    Wrap main in try/catch. On failure, log error and `process.exit(1)`. Same pattern as audit-questions.ts.
    On success, `process.exit(0)`.
