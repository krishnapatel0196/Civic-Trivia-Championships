**frontend/src/pages/Profile.tsx** — Full rewrite. Remove all identity management state and UI. Add dual-API fetch with partial failure handling.

Structure:

1. **State variables** — drastically simplified:
   - `triviaStats: TriviaStats | null` (from trivia backend)
   - `accountData: AccountProfile | null` (from accounts API via authService.fetchAccountProfile)
   - `loading: boolean`
   - `triviaError: string | null` (trivia backend failure)
   - `accountError: string | null` (accounts API failure)
   - `updatingTimer: boolean`
   - Remove ALL of: editingName, nameInput, savingName, nameError, showPasswordForm, currentPassword, newPassword, confirmPassword, savingPassword, passwordError, passwordSuccess, uploading, uploadError

2. **Data loading** — `useEffect` on mount:
   - Read `accessToken` from `useAuthStore.getState()`.
   - Use `Promise.allSettled([fetchTriviaStats(), authService.fetchAccountProfile(accessToken)])`.
   - For trivia result: if fulfilled, set `triviaStats`. If rejected, set `triviaError` to "Couldn't load game stats".
   - For account result: if fulfilled, set `accountData`. Check tier: if `accountData.tier === 'inform'`, redirect to `/signup` with `navigate('/signup', { replace: true })`. Also call `useAuthStore.getState().setDisplayName(accountData.display_name)`.
   - If rejected, set `accountError` to "Couldn't load account info".
   - If BOTH fail, set a general error state.
   - `setLoading(false)` in finally.

3. **Tier check** — after accountData loads:
   - If `accountData.tier === 'inform'`, redirect to `/signup` (Inform users can't see profile).
   - Do this check in the data loading callback, not in a separate effect.

4. **Render — Loading state**: keep existing spinner.

5. **Render — Hero section** (accounts data):
   - If `accountError`: show a yellow/amber warning banner: "Couldn't load account information. Some details may be unavailable." Then show whatever trivia data is available.
   - If `accountData` present:
     - Avatar component: use `accountData.avatar_url` for imageUrl, `accountData.display_name` for name. Remove `onUpload` prop (no more uploads from trivia).
     - Display name as h1 (large, bold, white) with tier badge INLINE next to it. Tier badge: small rounded label, color-coded:
       - `connected`: bg-teal-600/20 text-teal-400 border border-teal-500/30, text "Connected"
       - `empowered`: bg-purple-600/20 text-purple-400 border border-purple-500/30, text "Empowered"
     - Email below display name (slate-400).
     - "Manage your Empowered account" button: styled as a link/button with an arrow, opens `${ACCOUNTS_API_URL}` (or a known accounts platform URL) in a new tab. Use an anchor tag with `target="_blank" rel="noopener noreferrer"`. Import `ACCOUNTS_API_URL` from `../services/accountsApi` — or better, use a dedicated accounts platform URL env var if different from the API URL. For now, use `ACCOUNTS_API_URL` as the base (the user can update this later).
     - XP and Gems row:
       - XP: use `accountData.connected_profile?.xp ?? 0` with XpIcon (cyan).
       - Gems: use `accountData.connected_profile?.gem_balance ?? 0` with GemIcon (purple).

6. **Render — Stats section** (trivia data):
   - If `triviaError`: show an amber warning banner: "Couldn't load game stats. Try refreshing the page."
   - If `triviaStats` present: show games played, best score, overall accuracy — same layout as current.
   - Keep the empty state ("No games played yet!" with Play button) when `triviaStats.gamesPlayed === 0`.

7. **Render — Settings section**: keep Extended Time setting unchanged (trivia-specific).

8. **Removed UI** — do NOT include any of:
   - Name editing (inline edit input, save/cancel buttons)
   - Password change form
   - Avatar upload handler (keep Avatar component for display only, just remove onUpload)
   - Any edit icons next to the name
