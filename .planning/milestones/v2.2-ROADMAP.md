## Milestone Summary

**Key Decisions:**

- Scaffold brace scanner starts from ` = {` assignment, not TypeScript type annotation — eliminates false-positive brace counting from `Record<string, () => Promise<...>>` type
- `OfficeholderEntry` interface lives in `bloomington-in.ts` (sole `LocaleConfig` source of truth) — one canonical import path
- `buildOfficeholderBlock` wording is load-bearing ("name them SPECIFICALLY / do NOT write 'the current mayor'") — required for auto-seeder name-match
- `expiresAt IS NULL` guard in seeder query is mandatory — preserves manually corrected dates on re-runs
- `generateReplacement()` fully wrapped in try/catch — archive-first order ensures archival stands even on generation failure
- Leaderboard uses direct `supabaseAdmin.schema('connect')` access (no SECURITY DEFINER RPC needed — unlike XP history)
- Gem awards: plain try/catch (no withRetry) — contrast with XP pattern; `TRIVIA_GEMS_KEY` separate scoped key, not `TRIVIA_SERVICE_KEY`
- Santa Monica 13.1% → 19.8% expiring ratio: gap closure via Plan 03 insertion script; 1-per-officeholder rule limits achievable ratio
- Privacy-by-default on leaderboard: other users' display names blanked server-side

**Issues Resolved:**

- Scaffold Bug 2 (existed since Phase 57): brace scanner corruption now permanently fixed
- Manual expiresAt pass requirement: eliminated via structured officeholders field
- Gem award using deprecated Supabase RPC: migrated to accounts API
- No public leaderboard in CTC: shipped in Phase 67

**Issues Deferred:**

- `COLLECTION-PLAYBOOK.md` section 2 still documents scaffold workaround as active — doc cleanup deferred
- Live production gem smoke test: requires human verification with `TRIVIA_GEMS_KEY` on Render
- `empowered` tier badge not distinguishable from DB alone (resolves to `inform`/`connected` only) — accepted known limitation
- THIS WEEK tab and StickyYou out-of-top-25 require live data distribution to verify fully

**Technical Debt Incurred:**

- `add-smo-officeholder-questions.ts` is a one-shot script left in `/scripts/` — not harmful but adds clutter
- Leaderboard JS-side weekly XP aggregation: Supabase JS client lacks GROUP BY; full `xp_transactions` fetch for rolling 7 days may scale poorly at high user volumes
