# Milestone v1.9: Geographic Expansion

**Status:** ✅ SHIPPED 2026-03-03

**Total Plans:** 16

## Overview

Expand the playable collection set from 7 to 11 by activating two banked collections (Fremont CA, Norwich UK), adding two Massachusetts collections (Cambridge city, Massachusetts state), and two Texas collections (Plano city, Texas state). Eliminate hardcoded infrastructure debt that blocks future scaling.

## Milestone Summary

**Key Decisions:**

- Tier stored as text column (not enum) — simpler to add new tiers without DDL changes
- State config auto-discovery via dynamic import fallback — no registry to maintain, drop file in `state-configs/` and it works
- Mixed-durability pattern established: `texasStateFeatures` is the first locale config to instruct combined expiring + durable questions in one generation run
- Cambridge: Harvard/MIT strict limitation — universities cannot be primary civic subject in any question; Plan E City Manager (not Mayor) is the most important accuracy requirement
- Massachusetts: Governor's Council given 8 dedicated questions — one of MA's most surprising civic facts; general-court and governors-council each get own topic slug
- Plano: "Balloon Capital of Texas" designated as DURABLE TOPIC; corporate civic history (Frito-Lay, JCPenney, Toyota) scoped to civic significance only
- Texas: state-only rule strictly applied (city/regional landmarks archived); ERCOT cluster reduced to 3 best angles; expiresAt corrected for historical-fact dates
- scaffold-collection.ts step3 bug (inserts into type annotation line) documented and consistently worked around across phases 49, 50, 51, 52

**Issues Resolved:**

- Hardcoded COLLECTION_HIERARCHY fully eliminated from codebase
- State configs (indiana-state, california-state) now auto-discoverable via standard generate-locale-questions.ts workflow
- `curate-fremont-questions.ts` civic_trivia schema reference fixed (Drizzle ORM)
- `generateQuestions.ts` COLLECTION_NAMES stale values corrected (Indiana State → Indiana, California State → California) — caught in audit

**Issues Deferred:**

- scaffold-collection.ts step3 bug not patched at source — workaround documented and applied consistently
- Expiring question ratio enforcement (target 15–30%): only Texas (5%) and Cambridge MA (~50%) have expiring questions; standard not yet enforced — v2.0 backlog
- Auto-regeneration on expiry: `expirationSweep.ts` detects but does not replace — v2.0 backlog

**Technical Debt Incurred:**

- No VERIFICATION.md for Phase 49 (Cambridge MA) or Phase 52 (Texas State) — verification step skipped
- No 52-03-SUMMARY.md — plan executed (git commits 0c22dbf + 8adcd51 confirm activation) but summary document not written
