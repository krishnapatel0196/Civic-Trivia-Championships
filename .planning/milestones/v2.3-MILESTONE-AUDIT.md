- **Wager transition after tap-anywhere removal:** Wager phase reached via `NEXT_QUESTION` dispatch → `gameReducer.ts` → `final-announcement` → auto-transition to `START_WAGER` via `setTimeout`. No tap-anywhere dependency. Clean.

### Phase 70 Backend ↔ Frontend Contract

- `game.ts:397` calls `calculateProgression(results.totalCorrect, results.totalQuestions, results.totalScore)` — all three arguments from live session data.
- `GET /api/game/results/:sessionId` returns `progression.gemsEarned`; `ResultsScreen` consumes it correctly via `result.progression.gemsEarned`.

### Phase 71 Isolation

- `leaderboard.ts` has zero imports from `progressionService`, `gameReducer`, or any game component. Fully isolated. No regression risk from Phase 69/70 changes.
