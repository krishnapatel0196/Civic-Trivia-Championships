## Tech Debt

### Phase 63 — Scaffold Fix Documentation Gap

**Impact:** Non-blocking. Code fix is in place and correct. However:

1. `.planning/COLLECTION-PLAYBOOK.md` section 2 still documents the scaffold bug as active and instructs operators to run `git checkout -- backend/src/scripts/content-generation/generate-locale-questions.ts` after scaffolding
2. `MEMORY.md` (auto-memory, injected in every session) still lists the workaround with "Backlogged for fix"

Future agents and human operators following these docs will run an unnecessary and confusing workaround step. The workaround is harmless, but introduces confusion about whether the fix is in effect.

**Recommendation:** Update COLLECTION-PLAYBOOK.md section 2 to reflect that Scaffold Bug 2 is fixed as of Phase 63. Update MEMORY.md to remove the workaround instruction.

---

### Phase 66 — Live Production Smoke Test

**Impact:** Non-blocking on milestone completion. Code migration is complete and correct. However:

The final success criterion — "Gem awards land correctly in the accounts gem ledger (live production)" — requires human verification:

- Complete a full game as a Connected-tier user on the production deploy (civic-trivia-frontend.onrender.com)
- Confirm gem balance increases by the awarded amount (10 + correct answers)
- Confirm idempotency: replaying the same session does not double-award

This cannot be verified programmatically and requires `TRIVIA_GEMS_KEY` to be set on Render.
