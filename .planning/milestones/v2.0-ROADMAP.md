# Milestone v2.0: XP Integration

**Status:** ✅ SHIPPED 2026-03-08
**Phases:** 53–56
**Total Plans:** 10

## Overview

End-to-end XP integration with the Empowered Accounts platform. Connected players earn XP after every game (server-side award, score-proportional formula, idempotent), see their level and progress before and after games, and can review their full transaction history on the profile page.

## Milestone Summary

**Key Decisions:**

- `EMPOWERED_ACCOUNTS_API_URL` distinct from `EMPOWERED_ACCOUNTS_URL` — separate for independent configuration; EMPOWERED_ACCOUNTS_URL is used by checkAccountContext() and must not be renamed
- Idempotency key = `ctc-game-{sessionId}-{userId}` — sessionId is server-generated UUID, no separate gameId field needed
- `is_duplicate: true` treated as success — `xpResult.confirmed=true, isDuplicate=true`; no error, no retry
- XP formula: BASE_XP=50, VARIABLE_XP=150, score ratio to variable portion (range 50–200)
- Never-throw pattern: `awardPlatformXp()` mirrors `awardPlatformGems()` — withRetry, returns result object, never throws
- `console.warn` not `process.exit` for missing XP env vars — local dev must start cleanly
- `get_ctc_xp_history()` SECURITY DEFINER RPC is the established pattern for cross-schema `connect` data access from CTC backend
- 24h session TTL after XP award (vs 1h default) — only on post-award saveSession; submitAnswer keeps 1h
- Two-tab Profile tier-gated (`tierResolved && isConnected`) to prevent tab chrome flash
- `priorLevel` captured at game idle state; `levelUpShownRef` prevents double-trigger on re-renders

**Issues Resolved:**

- SECURITY DEFINER RPC pattern discovered: Accounts API proxy failed for connect schema data; Supabase RPC is correct cross-schema approach
- isDuplicate metadata input caused type noise: removed from input side, preserved in return type
- Session TTL edge case: 1h Redis expiry caused xp: null on repeat /results; 24h TTL closes the gap

**Issues Deferred:**

- EMPOWERED_ACCOUNTS_URL not in startup validation (minor ops hygiene — added to env.ts by Phase 56, post-audit)
- priorLevel capture race on Dashboard auto-start: XP data may not resolve before startGame() fires, keeping priorLevel null — pre-existing, not introduced in v2.0

**Technical Debt Incurred:**

- Human verification checklist (6 items) cannot be automated: idempotency live test, full Connected award flow, level-up overlay live execution, empty state, pagination at 21+ entries, isDuplicate label on known-duplicate row
