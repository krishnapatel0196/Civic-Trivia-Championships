# Milestone v1.7: Live Civic Intelligence

**Status:** ✅ SHIPPED 2026-02-27
**Phases:** 35-39
**Total Plans:** 10

## Overview

Make the trivia platform react to real civic events — election questions that appear when races are announced and expire on election day; a new UK city collection (Norwich, England, the platform's first non-US collection); and a quality rule that catches phone/address answers before they reach players. Phases 35, 37, 38, and 39 deliver the full election lifecycle pipeline; Phase 36 delivers the Norwich collection independently.

## Milestone Summary

**Key Decisions:**

- Admin-entered race data for v1.7 (no scrapers) — no reliable free API for US local elections; scraping deferred to v1.8+
- `claude-sonnet-4-6` hardcoded directly in `ElectionQuestionGenerator` — MODEL constant in anthropic-client.ts was outdated at claude-sonnet-4-5
- Collection slug passed explicitly as CLI/API parameter (not derived from jurisdiction string matching in generator)
- Election cron resolves collection slug via `collections.name = race.jurisdiction` DB lookup — cron has no slug context
- `GenerationBlockedError` from cron is idempotent skip — running cron twice for same race is expected behavior
- `lastCronRun` stored as module-level mutable let — no DB round-trip needed for admin banner
- `FollowupBlockedError` thrown when `followupGenerated=true` on enter-result call (idempotent 409 guard)
- `elc-term-{raceId}-{seq}` external ID format for current-term questions (distinct from campaign questions)
- `/classified` endpoint registered before `GET /election-races` to avoid Express `:id` param match on "classified"
- Awaiting Follow-up classification: `election_date < now AND followup_generated = FALSE`
- `GET /admin/collections` registered before `/collections/health` to maintain Express route order
- Jurisdiction validation returns 400 (not 404) — user input validation error
- DDL applied directly via pg client to bypass drizzle-kit interactive TTY prompt
- Follow-up questions have `expiresAt = NULL` — "Who won?" is a permanent historic fact
- `election_race_id` as direct FK on questions (not junction table) — each election question belongs to exactly one race
- `checkAddressPhone` scans answer options ONLY — not question.text or explanation; civic bodies legitimately reference addresses
- Advisory severity for address/phone rule — legitimate civic location questions can have address answers; flag for human review
- Norwich uses en-GB localeCode — first non-US collection
- Two-tier governance (City Council vs Norfolk County Council) encoded as critical accuracy requirement in voice guidance
- Norwich batchSize 15 (not 25) — smaller topic-focused batches across 8 topic categories
- Norwich themeColor #1B4332 (deep forest green) — visually distinct from all existing collection colors
- Norwich image sourced via Node.js HTTPS with Referer header (Wikimedia CDN rate-limited curl)
- Force-regenerate uses timestamp-suffixed externalIds (`elc-{raceId}-{ts36}-{seq}`) to avoid ON CONFLICT DO NOTHING silently skipping new questions
- getEndOfDayUTC anchors on local noon (not midnight) to correctly handle DST spring-forward dates

**Issues Resolved:**

- Phase 39 closed all 3 integration issues from initial v1.7 audit: admin endpoint, jurisdiction validation, collectionSlug override
- drizzle-kit interactive TTY prompt bypassed by applying DDL directly via pg client (safe, non-destructive workaround)
- `create-test-race.ts` TypeScript compilation error fixed (missing required candidate fields `party` and `incumbent`)

**Issues Deferred:**

- by-election and MP terminology absent from 117 Norwich questions (content gap, non-blocking, deferred to v1.8 content review)
- QUAL-04: admin manual review of audit-address-phone report (by design — human-gated process, no code gap)

**Technical Debt Incurred:**

- Norwich by-election/MP terminology gap — voice guidance specified these terms but generation produced 0/117 with either
- Election pipeline human verification items (DB schema on live Render, full lifecycle flow with live DB + Claude API) — require live env, non-blocking
