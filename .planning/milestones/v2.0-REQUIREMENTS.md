# Requirements: Civic Trivia Championship v2.0

**Defined:** 2026-03-05
**Core Value:** Make civic learning fun through game show mechanics — play, not study. No dark patterns, no guilt, no pressure.

## v2.0 Requirements

### XP Backend

- [x] **XP-01**: Backend awards XP via `POST /api/xp/award` after each game session for Connected players (server-to-server, service key auth, never exposed to browser)
- [x] **XP-02**: XP formula is score-proportional with a participation floor (BASE_XP=50, VARIABLE_XP=150, range 50–200)
- [x] **XP-03**: Each game session has a server-generated `gameId` used to construct the idempotency key (`ctc-game-<sessionId>-<userId>`)
- [x] **XP-04**: Non-Connected players (Inform/anonymous) are silently skipped — no award call is made
- [x] **XP-05**: `TRIVIA_SERVICE_KEY` and `EMPOWERED_ACCOUNTS_API_URL` env vars wired on CTC backend, documented, and validated at startup

### XP Start Screen

- [x] **XPS-01**: Connected players see their current level and XP progress bar before the game starts (`GET /api/xp/:userId`)
- [x] **XPS-02**: Non-Connected players see a "Link your account to earn XP" prompt on the start screen

### XP End Screen

- [x] **XPE-01**: Connected players see `+XP earned` with the game's awarded amount after game results
- [x] **XPE-02**: Level-up animation shown when award response `level > pre-game level`
- [x] **XPE-03**: Updated XP progress bar reflects new level position after the award
- [x] **XPE-04**: `is_duplicate: true` response shows a neutral "game already recorded" message instead of reward animation

### XP History

- [x] **XPH-01**: Profile page has an XP history panel showing past CTC game sessions and XP earned
- [x] **XPH-02**: History is paginated and fetched via `GET /api/xp/me/history` (Bearer JWT, Connected tier only)

## Out of Scope

| Feature | Reason |
|---------|--------|
| XP for non-game activities (e.g., flagging questions) | Platform defines XP sources; CTC is only authorized for `civic_trivia_championship_score` |
| Leaderboards | Research needed, may add later (from v1.x OUT OF SCOPE) |
| Daily XP caps | Platform has no daily cap; do not build this |
| Perfect-game bonus as separate transaction | Keep it simple for v2.0 — include in base formula |
