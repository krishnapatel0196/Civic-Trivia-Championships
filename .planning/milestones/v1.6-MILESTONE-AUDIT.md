## Bugs Found

### Bug 1 (High): `generateQuestions.ts` crashes for `--collection federal`
**Location:** `backend/src/scripts/generateQuestions.ts:133-144`
**Issue:** `questions.json` (Federal) is a bare JSON array. `loadCollectionData()` reads `jsonData.topics` and `jsonData.questions.map(...)` — both `undefined` → `TypeError`.
**Impact:** Any attempt to generate Federal questions crashes immediately. Not exercised in Phase 34 (Federal already at 114), but live in the codebase.
**Fix:** Check if `jsonData` is an array and handle accordingly, or add Federal to a "no-generate" list.

### Bug 2 (Medium): Auto-resolve success alert shows "undefined clusters"
**Location:** `backend/src/routes/admin.ts:1243` vs `frontend/src/pages/admin/DuplicateReviewPage.tsx:284`
**Issue:** API returns `{ clustersResolved: N }`, frontend reads `result.resolvedCount` → `undefined`.
**Impact:** Auto-resolve action succeeds (data mutation correct) but confirmation alert reads "Successfully resolved undefined clusters automatically."
**Fix:** Rename API field to `resolvedCount` or update frontend to read `clustersResolved`.
