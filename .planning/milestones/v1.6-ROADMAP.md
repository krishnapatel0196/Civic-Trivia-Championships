## Milestone Summary

**Key Decisions:**

- text-embedding-3-small (1536 dimensions) for balance of cost and accuracy in deduplication
- Disk-based embedding cache (.embedding-cache/) to avoid redundant API calls across scan runs
- p-limit(10) concurrency control for OpenAI rate limits
- Tiered similarity thresholds: exact >0.95, near-duplicate >0.85, possible >0.75
- Union-find clustering for transitive duplicate grouping (A≈B, B≈C → cluster{A,B,C})
- Collection hierarchy: Federal > State > City — parent blocks child, siblings allow cross-collection duplicates
- 30-second undo window for duplicate archive actions (Gmail/GitHub UX pattern)
- Auto-resolve threshold 0.90 for high-confidence automation
- Singleton DuplicateReviewService — undo state lost on server restart (acceptable for single-admin tool)
- Serial generation (not concurrent) for reliability over speed
- Output to JSON files before seeding — human review step preserved
- Accept source exhaustion as collection ceiling — no retry with inflated targets
- Archive higher-numbered externalId on conflict — preserve established question IDs with potential user history
- verify-phase34-final.ts exits 0 (PASS WITH ACCEPTED SHORTFALLS) not 1/FAIL for source-exhaustion shortfalls

**Issues Resolved:**

- 268 duplicate questions archived from 6 collections (human-reviewed, not automated)
- Federal collection cross-seeding caused many State/City duplicates — resolved via hierarchy policy
- Generation pipeline produced cross-run duplicates (DB baseline loaded at run start) — resolved by post-seed dedup check and archiving

**Issues Deferred:**

- generateQuestions.ts crashes for --collection federal (bare array format) — Federal at 114, not needed
- Auto-resolve success alert shows "undefined clusters" (API field mismatch) — functional but cosmetic
- DuplicateReviewService.ts has stale AdvancedFlag type — uses details field, real data uses severity/reason/evidence
- Frontend DuplicateSummary type declares fields never returned by API

**Technical Debt Incurred:**

- LA (88), Bloomington (75), Fremont (54) are source-exhausted — further generation unlikely without new source material
- Generation pipeline has cross-run duplicate risk (DB dedup baseline loaded at run start)
- 268 archived questions from Phase 32 curation could be selectively re-activated for gap closure (Phase 35 option)
