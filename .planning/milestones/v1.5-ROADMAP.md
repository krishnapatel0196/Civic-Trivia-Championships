# Milestone v1.5: Feedback Marks

**Status:** SHIPPED 2026-02-22
**Phases:** 27-30
**Total Plans:** 11

## Overview

Let authenticated players flag questions they dislike during gameplay, provide optional elaboration post-game, and give admins a review queue to triage flagged content — turning players into quality curators.

## Phases

### Phase 27: Backend Foundation & Inline Flagging

**Goal**: Establish flag data collection infrastructure and enable authenticated players to thumbs-down questions during gameplay without interrupting game flow.
**Depends on**: Phase 26 (v1.4 complete)
**Plans**: 2 plans (27-03 removed — verification done via checkpoint in 27-02)

Plans:
- [x] 27-01-PLAN.md — Backend infrastructure: database schema, feedback service, rate limiter, API endpoints
- [x] 27-02-PLAN.md — Frontend: FlagButton component, GameScreen integration, results screen display
- [x] ~~27-03-PLAN.md~~ — Removed (verification done via checkpoint in 27-02)

**Details:**
- PostgreSQL question_flags table with composite unique constraint
- Redis-based rate limiter (10 flags/15min, fails open)
- Transactional flag_count denormalization on questions table
- Idempotent flag operations with onConflictDoNothing
- FlagButton with amber fill toggle, optimistic UI, stopPropagation
- Rate limit detection disables button on 429 response

## Milestone Summary

**Key Decisions:**
- Rate limiter only on POST /flag, not DELETE — allow unrestricted unflagging
- Fail-open rate limiting on Redis errors — availability over abuse prevention
- Denormalize flag_count on questions table — avoid COUNT(*) queries
- No rate limiter on batch elaboration endpoint — infrequent post-game action
- Null for empty elaborations — database normalization
- Equal visual weight for Skip/Submit buttons — no dark patterns
- Elaboration after results, not before — user requested interactive flags on results screen
- pendingAction pattern — intercept Play Again/Home with elaboration before executing
- Hard delete flags on dismiss — simpler, no audit trail needed
- Custom undo toast vs library — 50-line custom hook avoids 3KB dependency
- AI suggestions limited to .gov/.edu/civic orgs for authoritative sources
- Null source URL for unrepairable links rather than keeping broken URLs
